## 1. AI / LLM

**AI 에이전트용 월러스 메모리 인프라 출시**
월러스(Wallrus)가 AI 에이전트가 여러 세션에 걸쳐 정보를 기억하고 활용할 수 있는 메모리 인프라를 공식 출시했습니다. 이 인프라는 클로드(Claude), 챗GPT(ChatGPT), 제미나이(Gemini) 등 주요 생성형 AI 서비스와 통합되어 에이전트의 지속적인 학습 및 복잡한 작업 수행 능력을 향상시킵니다.
**왜 주목할만한가:** 장기 기억은 AI 에이전트의 핵심 기능으로, 복잡한 ITS 시나리오(예: 장기 트래픽 패턴 학습, 사고 이력 기반 대응)에서 더욱 안정적이고 고도화된 에이전트 개발을 가능하게 합니다.

**엔비디아, 구글 디퓨전 젬마 가속화 및 SANA-WM 공개**
엔비디아가 구글의 오픈소스 이미지 생성 모델인 디퓨전 젬마(Diffusion Gemma)를 전격 가속화하여 로컬 AI 환경에서 최대 4배 빠른 처리 속도를 제공합니다. 이는 NVFP4 포맷 최적화를 통해 이루어졌습니다. 또한, 26억 파라미터 규모의 오픈소스 비디오 월드 모델 ‘SANA-WM’을 공개했습니다.
**왜 주목할만한가:** 엣지 디바이스에서의 ITS AI 모델(예: CCTV 영상 분석, 돌발 상황 감지) 추론 속도를 혁신적으로 개선할 수 있으며, SANA-WM은 복잡한 교통 상황 예측 및 시뮬레이션 모델 개발에 활용될 잠재력을 가집니다.

**오픈소스 LLM-DB 통합 표준, 모델 컨텍스트 프로토콜(MCP) 발표**
오픈소스 프로젝트인 모델 컨텍스트 프로토콜(MCP)이 LLM 기반 툴과 로컬 시스템, 내외부 API, 데이터 소스를 연결하는 표준으로 발표되었습니다. MCP는 PostgreSQL, MySQL, SQL 서버, 오라클 등 다양한 관계형 데이터베이스에 대한 LLM 액세스를 통합하여 자연어 기반의 데이터 조작을 가능하게 합니다.
**왜 주목할만한가:** ITS 데이터베이스(교통량, 사고 기록, 신호 정보 등)에 자연어 기반으로 질의하고 분석할 수 있는 LLM 애플리케이션 개발을 가속화하며, 복잡한 데이터 분석 및 리포팅 자동화에 기여할 수 있습니다.

## 2. 보안 & 취약점

**오라클 피플소프트 제로데이 취약점 악용 사례 발견 (CVE-2026-35273)**
해킹 조직 샤이니헌터스(ShinyHunters)가 오라클 피플소프트(Oracle PeopleSoft)의 제로데이 취약점인 CVE-2026-35273을 악용하여 대학 및 기업을 해킹한 정황이 포착되었습니다. 이 취약점은 인증 없이 원격에서 서버에 접근할 수 있게 합니다.
**왜 주목할만한가:** ITS 시스템이 내부적으로 오라클 솔루션이나 유사 엔터프라이즈 시스템을 사용하는 경우, 즉각적인 패치 적용 및 시스템 전반의 보안 감사가 필요함을 시사합니다.

**마이크로소프트, 다수 취약점 및 제로데이 5건 긴급 패치 (CVE-2026-45585 등)**
마이크로소프트가 200개 이상의 취약점을 수정하는 긴급 업데이트를 발표했으며, 이 중 5건은 제로데이 취약점으로 확인되었습니다. 특히 윈도우 디스크 암호화 기능인 비트로커(BitLocker)를 우회할 수 있는 '옐로키(YellowKey)' 취약점(CVE-2026-45585) 2건이 포함되었습니다.
**왜 주목할만한가:** ITS 운영 환경의 윈도우 서버 및 클라이언트 시스템에 대한 신속한 업데이트가 필수적이며, 데이터 암호화 솔루션의 우회 가능성에 대한 주의가 요구됩니다.

**구글 크롬 V8 엔진 제로데이 취약점 긴급 업데이트 권고**
구글 크롬 브라우저의 V8 엔진에서 실제 공격에 악용된 제로데이 취약점이 확인되어 긴급 업데이트가 권고되었습니다. 구체적인 CVE ID는 공개되지 않았으나, 사용자들의 즉각적인 브라우저 업데이트가 필요합니다.
**왜 주목할만한가:** ITS 관제 시스템이나 관련 웹 기반 애플리케이션에서 크롬 브라우저를 사용하는 경우, 시스템 접근 권한 탈취 등 심각한 보안 위협으로 이어질 수 있으므로 즉시 업데이트해야 합니다.

## 3. 개발 도구 & 프로그래밍

**GitHub 코파일럿, 개발 사이클 시간 최대 40% 단축 입증**
깃허브 코파일럿(GitHub Copilot)이 개발자의 코드 작성 시간을 단축시키고 생산성을 높여 개발 사이클을 최대 40%까지 줄일 수 있음을 실제 사례를 통해 입증했습니다. 이는 개발자가 더 적은 노력으로 더 빠르게 코드를 작성하도록 돕는 AI 코딩 어시스턴트의 효과입니다.
**왜 주목할만한가:** ITS 개발 프로젝트에서 반복적인 코딩 작업이나 신규 기능 구현 시 코파일럿을 활용하여 개발 속도를 높이고, 개발자들이 핵심적인 문제 해결에 집중할 수 있도록 지원합니다.

**OpenAI, LLM 토큰 가격 인하로 AI 코딩 비용 효율성 제고**
오픈AI가 기업 고객 유치를 위해 앤트로픽(Anthropic)과의 경쟁 속에서 LLM 사용료(토큰 가격)를 대폭 인하했습니다. 이는 코덱스(Codex)와 같은 OpenAI의 AI 코딩 도구를 포함한 전반적인 LLM 활용 비용을 낮춰 개발자들이 AI 기반 솔루션을 보다 경제적으로 개발하고 배포할 수 있도록 합니다.
**왜 주목할만한가:** ITS 분야에서 AI 기반 코드 생성, 테스트 자동화, 문서화 등에 LLM을 활용하는 비용 부담이 줄어들어, AI 도입 장벽을 낮추고 더 많은 AI 코딩 도구의 활용을 촉진합니다.

**GitHub 플랫폼 보안 강화 권고: 인증 토큰 관리 중요성 부각**
데이원컴퍼니 등 국내 기업에서 깃허브(GitHub) 플랫폼 내 인증 토큰이나 접근 권한 정보가 외부에 노출되어 내부 시스템 침해로 이어지는 사례가 발생했습니다. 이는 개발자들이 소스코드를 저장하고 관리하는 과정에서 보안에 대한 경각심을 높일 필요성을 보여줍니다.
**왜 주목할만한가:** ITS 프로젝트의 소스코드가 GitHub와 같은 플랫폼에 관리되는 경우, 인증 토큰, API 키, 민감 정보 등의 안전한 관리와 GitHub Advanced Security와 같은 보안 기능 활용이 필수적입니다.

3.  **GitHub Advanced Security 기능 검토 및 적용:** GitHub 리포지토리의 인증 토큰 노출 방지를 위해 GitHub Advanced Security (CodeQL, Dependabot 등) 기능을 검토하고, 사내 개발 워크플로우에 통합하여 잠재적 보안 취약점을 사전에 탐지하고 수정하는 방안을 모색합니다. GitHub 공식 문서 (`https://docs.github.com/en/advanced-security`)를 참고합니다.

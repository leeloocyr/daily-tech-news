## 1. AI / LLM

**월러스(Wallrus) AI, 에이전트용 장기 메모리 인프라 '월러스 메모리(Wallrus Memory)' 공개**
월러스 AI가 AI 에이전트가 여러 세션에 걸쳐 컨텍스트를 유지하고 복잡한 추론을 수행할 수 있도록 돕는 장기 메모리 인프라 '월러스 메모리'를 공식 출시했다. 이 솔루션은 클로드(Claude), 챗GPT(ChatGPT), 제미나이(Gemini) 등 주요 생성형 AI 서비스와 통합 가능하며, AI 에이전트의 일관성과 효율성을 높인다.
**왜 주목할만한가:** ITS에서 다단계 의사결정이나 장기적인 교통 상황 분석을 수행하는 AI 에이전트를 개발할 때, 에이전트가 이전 대화나 이벤트 데이터를 기억하고 활용하도록 하는 데 필수적인 기능으로, 복잡한 ITS 시나리오 처리 능력 향상에 기여한다.

**구글 딥마인드, 오픈소스 실험 모델 '디퓨전젬마(DiffusionGemma)' 공개**
구글 딥마인드가 텍스트 디퓨전 방식을 적용한 오픈소스 실험 모델 '디퓨전젬마(DiffusionGemma)'를 6월 10일(현지시간) 공개했다. 이 모델은 기존 LLM처럼 토큰을 한 개씩 순차적으로 생성하지 않고, 텍스트 전체를 동시에 생성하여 텍스트 생성 속도를 획기적으로 향상시킬 잠재력을 가지고 있다.
**왜 주목할만한가:** 실시간 교통 정보 분석, 돌발 상황 예측 및 즉각적인 정보 제공이 중요한 ITS 환경에서 LLM의 추론 및 생성 속도 개선은 서비스 반응성을 극대화할 수 있으므로, 해당 모델의 성능 특성을 분석하고 ITS 데이터에 적용 가능성을 탐색할 필요가 있다.

**엔비디아와 네이버, 오픈소스 LLM '네모트론(NemoTron)' 개발 생태계 참여 강화**
네이버가 엔비디아의 개방형 오픈소스 대형 언어 모델(LLM)인 '네모트론(NemoTron)' 개발 및 활용 생태계에 참여하며 글로벌 기술 협력을 강화한다. 이는 미스트랄AI, 퍼플렉시티 등 여러 글로벌 AI 기업들과 함께 엔비디아의 차세대 고성능 LLM 개발에 기여하는 공동 노력의 일환이다.
**왜 주목할만한가:** 대규모 오픈소스 LLM 생태계에 한국 기업이 참여하고 기술 기여를 확대함에 따라, ITS 개발자들은 더 강력하고 유연한 오픈소스 LLM 자원을 활용하여 도메인 특화 모델을 구축하고 엔비디아 GPU 기반 최적화된 추론 환경을 활용할 기회를 얻게 된다.

**SK텔레콤, 지식형 LLM '에이닷 엑스(A.X) 4.0' 오픈소스로 공개**
SK텔레콤이 자체 개발한 지식형 LLM '에이닷 엑스(A.X) 4.0'을 오픈소스로 공개했다. 이 모델은 특정 도메인 지식 학습에 최적화되어 있으며, 기업 및 개발자들이 자체 서비스에 쉽게 통합하고 활용할 수 있도록 설계되었다.
**왜 주목할만한가:** 교통 정책, 도로 규정, 지역별 특수성 등 ITS 분야의 방대한 전문 지식을 효과적으로 학습하고 추론할 수 있는 지식형 LLM의 오픈소스 공개는 ITS 개발자들이 자사 서비스에 도메인 특화 AI 기능을 빠르고 효율적으로 구현하는 데 중요한 기반을 제공한다.

## 2. 보안 & 취약점

**시스코 카탈리스트 SD-WAN 매니저 제로데이 취약점 (CVE-2026-20245) 실제 공격 악용 경고** ⭐
시스코(Cisco)는 시스코 카탈리스트 SD-WAN 매니저(Cisco Catalyst SD-WAN Manager)에서 발견된 고위험 제로데이 취약점 CVE-2026-20245가 실제 공격에 악용되고 있다고 6월 4일 경고했다. 이 취약점은 원격 공격자가 인증 없이 루트 권한을 상승시켜 시스템을 완전히 제어할 수 있도록 하며, 아직 보안 패치가 공개되지 않았다.
**왜 주목할만한가:** SD-WAN은 차세대 ITS 인프라에서 스마트 도로망 및 교통 관제 센터 간의 네트워크 연결성을 담당할 수 있어, 이 치명적인 취약점은 ITS 시스템 전체의 운영 안정성과 보안에 심각한 위협이 될 수 있으므로 즉각적인 경계와 대응 계획 수립이 필수적이다.

**구글 크롬 V8 엔진 제로데이 취약점 (CVE-2026-2441 등) 긴급 패치 및 실제 공격 확인**
구글은 올해 들어 여러 차례에 걸쳐 크롬(Chrome) V8 자바스크립트 엔진에서 발견된 제로데이 취약점들(CVE-2026-2441, CVE-2026-3909, CVE-2026-3910, CVE-2026-5281 등)에 대한 긴급 패치를 배포했다. 이 취약점들은 모두 실제 공격에 악용된 정황이 확인되었으며, 사용자들은 즉시 크롬 브라우저를 최신 버전으로 업데이트해야 한다.
**왜 주목할만한가:** ITS 개발자들은 대시보드, 관리 툴, 혹은 개발 환경에서 웹 브라우저를 광범위하게 사용하므로, 브라우저의 제로데이 취약점은 개발자 워크스테이션을 통한 내부 시스템 침투 경로가 될 수 있다. 모든 개발자는 최신 보안 패치를 적용하여 잠재적인 위협으로부터 자신과 프로젝트를 보호해야 한다.

**개발자 대상 GitHub 계정 및 토큰 유출 경고, 표적 공격 증가**
최근 티빙, 패스트캠퍼스 등 국내외 기업에서 GitHub 계정 자격 증명 및 API 토큰 유출 사례가 빈번하게 발생하고 있으며, 북한 해커 조직(라자루스 등)은 가짜 코딩 테스트를 가장하여 개발자 PC에 악성코드를 유포하는 표적 공격을 감행하고 있다. 이러한 공격은 소스코드 탈취, 민감 정보 유출 및 시스템 장악으로 이어질 수 있다.
**왜 주목할만한가:** ITS 개발 환경에서 소스코드 관리 및 협업에 필수적인 GitHub 계정 및 토큰의 보안은 프로젝트의 생명과 직결된다. 강력한 2단계 인증, 접근 권한 최소화, 주기적인 토큰 교체, 그리고 피싱 및 사회공학적 공격에 대한 개발자들의 보안 의식 강화와 교육이 시급하다.

## 3. 개발 도구 & 프로그래밍

**GitHub Copilot, Git 커밋 메시지 자동 생성 및 성능 최적화 제안 기능 도입**
GitHub Copilot이 최근 업데이트를 통해 개발자가 변경한 코드를 분석하여 Git 커밋 메시지를 자동으로 제안하고 생성하는 기능을 정식 도입했다. 또한, 특정 프로그래밍 언어(예: Python, Java)에 대해 잠재적인 성능 병목 지점을 식별하고, 해당 부분을 최적화할 수 있는 코드 스니펫을 제안하는 기능이 베타 버전으로 추가되었다.
**왜 주목할만한가:** 반복적인 커밋 메시지 작성 시간을 단축하고 일관성 있는 커밋 이력을 관리하는 데 도움을 주며, 특히 실시간 처리 및 대규모 데이터 처리가 중요한 ITS 애플리케이션 개발 시 성능 최적화된 코드를 작성하는 데 기여하여 개발 생산성과 코드 품질을 동시에 향상시킬 수 있다.

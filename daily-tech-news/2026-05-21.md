## 1. AI / LLM

**Anthropic, Claude 4 발표 및 강력한 성능 개선**
Anthropic은 자사 최신 대규모 언어 모델인 Claude 4를 공식 출시했습니다. Claude 4는 MMLU 92.5%, HumanEval 90.1%를 기록하며 기존 Claude 3 Opus 대비 약 5%p 이상의 성능 향상을 보였습니다. 특히, 200K 토큰의 긴 컨텍스트 윈도우를 기본 지원하며, 시각 및 오디오 모달리티 처리 능력도 대폭 강화되어 복잡한 멀티모달 프롬프트에 대한 이해도가 크게 향상되었습니다.
**왜 주목할만한가:** ITS 개발자는 Claude 4의 향상된 컨텍스트 처리 능력과 멀티모달 기능을 활용하여 교통 데이터 분석, 시뮬레이션 결과 요약, CCTV 영상 분석 스크립트 자동 생성 등 복잡한 태스크를 더욱 정교하게 수행할 수 있습니다.

**Google, Gemini CLI 정식 출시로 명령줄 AI 코딩 지원 확대**
Google이 Gemini 모델을 활용한 공식 CLI(Command Line Interface) 도구를 정식 출시했습니다. 이 CLI는 개발자가 터미널 환경에서 직접 Gemini 모델에 접근하여 코드 생성, 디버깅, 스크립트 작성 등 다양한 개발 작업을 수행할 수 있도록 지원합니다. 초기 벤치마크에서는 Bash 스크립트 생성 시 HumanEval-Shell 78%의 성공률을 기록했으며, Python 및 JavaScript 코드 생성에도 최적화되어 있습니다.
**왜 주목할만한가:** 명령줄 환경에 익숙한 ITS 개발자들은 별도의 IDE나 웹 인터페이스 없이도 효율적으로 AI의 도움을 받아 시스템 관리 스크립트 작성, 데이터 전처리 명령어 생성 등 반복적인 작업을 자동화하고 개발 생산성을 높일 수 있습니다.

**DeepSeek, DeepSeek-VLM 2.0 오픈소스 공개 및 멀티모달 LLM 경쟁 심화**
중국의 DeepSeek AI가 8B 및 23B 파라미터 규모의 비전 언어 모델(VLM)인 DeepSeek-VLM 2.0을 오픈소스로 공개했습니다. 이 모델은 다양한 멀티모달 벤치마크에서 Llama 3 8B 기반 VLM들을 능가하는 성능을 보였으며, 특히 OCR(광학 문자 인식) 및 그래프 이해 능력에서 강점을 가집니다. Apache 2.0 라이선스 하에 배포되어 상업적 사용이 가능합니다.
**왜 주목할만한가:** ITS 분야에서 CCTV 영상, 도로 표지판, 대시보드 데이터 등 다양한 시각 정보를 처리해야 하는 개발자들에게 DeepSeek-VLM 2.0은 비용 효율적인 대안을 제공하며, 온프레미스 또는 엣지 디바이스 환경에서의 특정 비전 태스크 모델 구축에 활용될 수 있습니다.

⭐ **디노티시아, AI 에이전트 전용 오픈소스 지식 플랫폼 ‘AKB’ 공개**
디노티시아가 AI 에이전트의 효율적인 지식 활용을 위한 오픈소스 지식 플랫폼인 'AKB(Agent Knowledge Base)'를 GitHub에 공개했습니다. AKB는 기업 내부 문서 및 외부 데이터를 통합 관리하고, 에이전트가 정확하고 최신 정보를 기반으로 의사결정을 내릴 수 있도록 벡터 데이터베이스, 그래프 DB, 시맨틱 검색 기능을 통합 제공합니다. RAG(Retrieval Augmented Generation) 아키텍처에 최적화되어 있습니다.
**왜 주목할만한가:** ITS에서 자율 에이전트 시스템을 구축하려는 개발자들에게 AKB는 교통 상황 데이터, 정책 문서, 장비 매뉴얼 등 방대한 정보를 에이전트가 신뢰성 있게 접근하고 활용할 수 있는 핵심 인프라를 제공하여, 지능형 교통 관제 및 대응 시스템 개발을 가속화합니다.

## 2. 보안 & 취약점

**Palo Alto Networks PAN-OS에서 인증 우회 제로데이 취약점 (CVE-2026-0300) 발견**
Palo Alto Networks의 방화벽 운영체제인 PAN-OS의 'User-ID 인증 포털'에서 심각한 인증 우회 제로데이 취약점인 CVE-2026-0300이 발견되었습니다. 이 취약점은 공격자가 특정 API 호출을 통해 인증 절차 없이 시스템에 접근할 수 있게 하며, CVSS 점수 9.8 (Critical)로 평가됩니다. 현재 긴급 패치가 배포 중이며, 즉각적인 업데이트가 권고됩니다.
**왜 주목할만한가:** ITS 인프라의 핵심 보안 장비인 방화벽의 취약점은 전체 교통 시스템의 통제권 탈취 및 데이터 유출로 이어질 수 있으므로, ITS 개발 및 운영팀은 해당 패치 적용 여부를 최우선으로 확인하고 대비해야 합니다.

**Windows 클라우드 파일 드라이버의 제로데이 취약점 (CVE-2025-62221) PoC 공개**
Windows 클라우드 파일 드라이버(MiniPlasma)에서 발견된 제로데이 취약점 CVE-2025-62221에 대한 개념증명(PoC) 코드가 공개되었습니다. 이 취약점은 공격자가 관리자 권한을 획득하거나 시스템 크래시를 유발할 수 있으며, 이전에 패치되었다고 알려진 결함(CVE-2024-XXXXX)이 완전하게 수정되지 않았음이 드러났습니다. Microsoft는 추가 패치를 준비 중입니다.
**왜 주목할만한가:** ITS 시스템이 Windows 기반의 서버나 엣지 디바이스를 사용하는 경우, 이 취약점은 핵심 서비스의 중단이나 민감 데이터 접근으로 이어질 수 있어, 시스템 관리자와 개발자는 해당 PoC 코드의 확산에 대비하고 추가 보안 업데이트를 주시해야 합니다.

**워드프레스 플러그인에서 인증 우회 파일 삭제 제로데이 (CVE-2026-40769, CVSS 9.1) 발견**
인기 있는 워드프레스 플러그인에서 인증 절차 없이 임의의 파일을 삭제할 수 있는 치명적인 제로데이 취약점 CVE-2026-40769 (CVSS 9.1)가 옵시아(Opsia) 연구팀에 의해 단독 발굴되었습니다. 이 취약점은 웹 서버에 저장된 중요한 설정 파일이나 사용자 데이터를 무단으로 삭제하여 서비스 거부 공격을 유발할 수 있습니다.
**왜 주목할만한가:** ITS 관련 정보 제공 웹사이트나 관리 포털이 워드프레스 기반으로 구축된 경우, 해당 취약점은 서비스 가용성에 직접적인 위협이 됩니다. 개발자는 사용 중인 플러그인 목록을 점검하고, 보안 공지를 확인하여 즉각적인 업데이트 또는 대체 조치를 강구해야 합니다.

## 3. 개발 도구 & 프로그래밍

**Automation Anywhere, 엔터프라이즈급 자율 에이전트 거버넌스 도구 'EnterpriseClaw' 출시**
Automation Anywhere가 시스코, 엔비디아, 옥타, OpenAI와의 협력을 통해 엔터프라이즈급 자율 에이전트 오케스트레이션 도구 'EnterpriseClaw'를 출시했습니다. 이는 기존 오픈소스 'OpenClaw'의 혼돈을 해결하고, 대규모 에이전트 배포 및 관리, 보안, 규정 준수를 위한 통합 거버넌스 프레임워크를 제공합니다. 기업용 워크플로우 자동화에 최적화되어 있습니다.
**왜 주목할만한가:** 복잡한 ITS 환경에서 수많은 AI 에이전트가 상호작용하며 자율적으로 작업을 수행해야 할 때, EnterpriseClaw는 에이전트의 행동을 일관성 있게 관리하고, 보안 정책을 적용하며, 전체 시스템의 안정성과 신뢰성을 확보하는 데 필수적인 도구가 될 수 있습니다.

**Zoom AI 컴패니언, 개발자용 AI 코딩 환경 통합 플러그인 출시**
Zoom은 자사 AI 컴패니언 기능을 확장하여 개발자를 위한 AI 코딩 환경 통합 플러그인을 출시했습니다. 이 플러그인은 Zoom 미팅 요약, 대화 내용 분석 등을 활용하여 개발자들이 논의한 내용을 바탕으로 코드 스니펫 제안, API 문서 검색, 협업 코드 수정 가이드 등을 실시간으로 제공합니다. Visual Studio Code 및 JetBrains IDEs와 연동됩니다.
**왜 주목할만한가:** ITS 개발팀이 화상회의를 통해 프로젝트를 논의하고 코딩 작업을 진행할 때, Zoom AI 컴패니언 플러그인은 회의록 기반의 코드 자동 생성 및 문서화 지원을 통해 팀의 개발 생산성과 협업 효율성을 크게 향상시킬 수 있습니다.

**GitHub, 코드 저장소 해킹 인정 및 보안 강화 조치 발표**
GitHub는 최근 발생한 해킹 공격으로 약 3,800개의 내부 저장소 데이터가 탈취되었음을 공식 인정하고, 즉각적인 보안 강화 조치를 발표했습니다. 이번 공격은 개발자들이 사용하는 Visual Studio Code의 특정 확장 프로그램 취약점을 통해 이루어진 것으로 파악됩니다. GitHub는 모든 사용자에게 2FA (Two-Factor Authentication) 사용을 강력히 권고하며, API 토큰 및 SSH 키의 정기적인 검토를 요청했습니다.
**왜 주목할만한가:** ITS 프로젝트의 핵심 코드와 개발 문서는 GitHub와 같은 코드 저장소에 보관되는 경우가 많습니다. 이번 사건은 개발 프로세스 전반의 보안 취약점을 다시 한번 경고하며, 모든 ITS 개발자는 자신의 개발 환경, 계정 보안, 그리고 사용 중인 도구의 보안 패치 상태를 철저히 점검해야 함을 시사합니다.

## 2. Claude / Anthropic

### Claude Advisor Tool — 에이전트 비용 혁신 (Public Beta)
Anthropic이 **Advisor Tool**을 Public Beta로 출시했다. 핵심 아이디어: **저렴한 모델(Sonnet/Haiku)이 실행하고, 어려운 판단만 Opus에게 물어보는 패턴**.

**동작 방식:**
- **Executor (실행자):** Sonnet 4.6 또는 Haiku 4.5가 도구 호출, 결과 처리, 응답 생성
- **Advisor (조언자):** Opus 4.6이 복잡한 판단/전략만 담당. 상담 시 400-700 토큰만 생성
- 하나의 API 호출 안에서 실행자가 자동으로 조언자에게 에스컬레이션

**벤치마크 결과:**
| 구성 | SWE-bench ML | 비용 변화 |
|---|---|---|
| Sonnet 단독 | 72.1% | 기준 |
| **Sonnet + Opus Advisor** | **74.8%** | **-11.9%** |
| Haiku BrowseComp 단독 | 19.7% | 기준 |
| **Haiku + Opus Advisor** | **41.2%** | 대폭 절감 |

**왜 주목할만한가:** "비싼 모델 성능은 필요하지만 비용은 줄이고 싶다"는 모든 팀의 고민에 대한 직접적인 해답이다. Sonnet으로 대부분 처리하고 어려운 것만 Opus가 개입하면 **성능은 올라가면서 비용은 11.9% 줄어든다**. 에이전트 파이프라인을 설계할 때 반드시 고려해야 할 패턴.

> ⭐ CCTV 분석 에이전트에서 "일반적인 프레임은 저렴한 모델이 처리하고, 사고/이상 상황이 의심될 때만 고성능 모델이 판단"하는 계층형 구조를 API 하나로 구현 가능. ROI 자동 세팅에서도 "일반 도로는 Sonnet, 복잡한 교차로/JC는 Opus에게 물어보기" 패턴이 가능.

- [Anthropic Docs](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool) | [BuildFastWithAI](https://www.buildfastwithai.com/blogs/anthropic-advisor-strategy-claude-api)

### OpenAI도 Axios 공급망 공격 영향 받았다
OpenAI가 GitHub Actions 워크플로우에서 macOS 앱 서명 시 **악성 Axios 라이브러리(3/31 공격분)를 다운로드**했다고 밝혔다. OpenAI는 "사용자 데이터 접근, 시스템/지적재산 침해, 소프트웨어 변조 증거는 없다"고 해명. 하지만 **OpenAI조차 당했다는 사실**이 공급망 공격의 심각성을 보여준다.

**왜 주목할만한가:** 세계 최대 AI 기업도 npm 공급망 공격에서 자유롭지 못했다. CI/CD 파이프라인의 자동 의존성 설치가 얼마나 위험한지 다시 한번 경고하는 사건.

## 3. 개발 도구 & 트렌드

### Gemini 3.1 Pro + Gemma 4 하이브리드 에이전트 워크플로우
Vertex AI에서 **Gemini 3.1 Pro (클라우드) + Gemma 4 (엣지)**를 조합한 하이브리드 에이전트 워크플로우를 구축하는 가이드가 공개됐다. 복잡한 판단은 클라우드에서, 실시간 추론은 엣지에서 처리하는 아키텍처.

**왜 주목할만한가:** Claude Advisor Tool과 비슷한 개념이지만 Google 진영에서는 **클라우드+엣지 분리**로 접근하고 있다. ITS처럼 실시간 + 분석이 모두 필요한 시스템에 특히 잘 맞는 패턴.

> ⭐ CCTV 엣지(Gemma 4)에서 실시간 1차 판단 → 이상 감지 시 클라우드(Gemini Pro)로 정밀 분석. 이게 ROI 자동 세팅 + 실시간 정체 모니터링의 이상적인 아키텍처일 수 있다.

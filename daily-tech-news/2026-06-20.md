## 1. AI / LLM

**Anthropic Claude 4.5 공개: 100만 토큰 컨텍스트와 강화된 추론 능력**
Anthropic이 자사의 최신 대규모 언어 모델인 Claude 4.5를 정식 출시했습니다. 이 모델은 최대 100만 토큰의 컨텍스트 윈도우를 지원하여 이전 모델 대비 2배 확장된 장문 이해 및 생성 능력을 제공합니다. MMLU(Massive Multitask Language Understanding) 벤치마크에서 92.5%, HumanEval 코딩 벤치마크에서 88.2%를 기록하며 SOTA(State-of-the-Art) 성능을 경신했습니다. Claude 4.5는 특히 복잡한 멀티모달 추론에서 강점을 보이며, 시각 정보와 텍스트 설명을 통합하여 복잡한 ITS 시나리오 분석에 활용될 수 있습니다.
**왜 주목할만한가:** 방대한 컨텍스트 처리 능력은 교통 흐름 데이터, CCTV 영상 로그, 센서 데이터를 종합 분석하는 ITS 시스템 개발에 혁신적인 변화를 가져올 수 있습니다. 장문의 정책 문서나 복잡한 코드베이스 이해에도 유리합니다.

**Walrus Memory, AI 에이전트용 영구 메모리 인프라 출시**
월러스(Walrus)가 AI 에이전트가 여러 세션에 걸쳐 정보를 기억하고 학습할 수 있도록 돕는 영구 메모리 인프라를 공식 출시했습니다. 이 서비스는 클로드(Claude), 챗GPT(ChatGPT), 제미나이(Gemini) 등 주요 생성형 AI 서비스와 통합 가능하며, 에이전트가 장기 기억을 유지하면서 복잡하고 다단계적인 작업을 수행할 수 있도록 설계되었습니다. 특히, 에이전트가 과거의 상호작용과 경험을 바탕으로 미래 의사결정을 개선하는 데 필수적인 기능을 제공합니다.
**왜 주목할만한가:** ITS 분야에서 교통 관제, 예측, 최적화 에이전트 개발 시, 과거의 교통 상황이나 돌발 상황 처리 경험을 기억하고 학습하는 데 핵심적인 역할을 할 수 있습니다. 이는 에이전트의 자율성과 효율성을 크게 향상시킬 것입니다.

⭐ **버클리 대학, 7B 파라미터 오픈소스 VLA 모델 'OpenVLA' 공개**
버클리 대학이 Vision-Language-Action (VLA) 모델인 OpenVLA를 70억 파라미터 규모의 오픈소스로 공개했습니다. OpenVLA는 "식탁 위의 사과를 바구니에 담아줘"와 같은 자연어 명령을 시각 정보와 로봇 동작으로 연결하여 수행하는 데 특화되어 있습니다. 기존의 T5-Base나 Llama-2 같은 LLM을 기반으로 하여, 이미지 인식 및 로봇 제어 기능이 통합되었습니다. 이 모델은 로봇이 실제 환경에서 복잡한 작업을 학습하고 수행하는 데 필요한 핵심 기술을 제공합니다.
**왜 주목할만한가:** ITS 환경에서 자율주행 차량, 드론, 혹은 로봇 기반의 교통 관리 시스템 개발 시, 자연어 지시를 받아 물리적 환경에서 복잡한 작업을 수행하는 데 직접적으로 활용될 수 있습니다. 7B 파라미터 규모는 엣지 디바이스 배포 가능성을 높입니다.

**오픈소스 LLM, 상용 모델 벤치마크 격차 최소화: DeepSeek V3, Llama 3.5 약진**
최근 딥시크(DeepSeek) V3와 메타(Meta)의 라마(Llama) 3.5 등 오픈소스 거대언어모델(LLM)이 주요 벤치마크에서 독점 상용 모델과의 성능 격차를 크게 줄였습니다. 특히, DeepSeek V3는 MMLU 벤치마크에서 89.1%, HumanEval에서 85.5%를 달성하며 Llama 3.5의 87.9% (MMLU) 및 82.1% (HumanEval)를 넘어섰습니다. 이는 오픈소스 커뮤니티의 활발한 개발과 혁신이 빠르게 진행되고 있음을 보여주며, 비용 효율적인 대안을 제시하고 있습니다.
**왜 주목할만한가:** ITS 솔루션 구축 시 비용 및 데이터 주권 문제로 상용 모델 도입이 어려운 경우, 고성능 오픈소스 LLM은 강력한 대안이 됩니다. 자체 인프라에 배포하여 특정 교통 도메인 데이터로 파인튜닝하는 전략을 고려할 수 있습니다.

## 2. 보안 & 취약점

**Microsoft Defender 제로데이 취약점 'RoguePlanet'(CVE-2026-50656) 발견**
마이크로소프트 디펜더(Microsoft Defender)에서 'RoguePlanet(로그플래닛)'이라는 코드명의 제로데이 취약점 CVE-2026-50656이 확인되었습니다. 이 취약점은 공격자가 SYSTEM 권한으로 임의 코드를 실행할 수 있게 하며, 공통 취약점 평가 기준(CVSS) 점수는 7.8로 심각한 수준입니다. 마이크로소프트는 현재 해당 취약점에 대한 패치를 긴급 배포 중입니다.
**왜 주목할만한가:** ITS 시스템은 대규모 서버와 엣지 디바이스에서 Microsoft 운영체제를 사용하는 경우가 많으므로, 이 취약점은 시스템 전체에 심각한 보안 위협을 초래할 수 있습니다. 즉각적인 패치 적용이 필수적입니다.

**Cisco NX-OS 제로데이 취약점 (CVE-2024-20399)을 이용한 10년 장기 침투 작전 공개**
중국 해킹 조직이 시스코 엔엑스오에스(Cisco NX-OS) 제로데이 취약점 CVE-2024-20399를 지난 10년간 악용하여 시스코 넥서스(Cisco Nexus) 스위치에 악성코드를 심고 장기적인 침투 작전을 펼쳤음이 최근 분석을 통해 공개되었습니다. 이 취약점은 네트워크 인프라의 핵심 장비에 대한 통제권을 탈취하는 데 사용되었습니다.
**왜 주목할만한가:** ITS 인프라는 시스코 같은 주요 네트워크 장비를 광범위하게 사용합니다. 이러한 핵심 인프라 장비의 제로데이 취약점은 전체 교통망의 마비나 데이터 유출로 이어질 수 있으므로, 네트워크 보안에 대한 지속적인 모니터링과 공급망 보안 검토가 중요합니다.

**LLM 에이전트 대상 'Poisoned Prompt Injection' 공격 취약성 증가**
최근 'PromptGuard Framework 1.0' 보고서에 따르면, 복잡한 작업을 수행하는 LLM 기반 에이전트 시스템에 대한 'Poisoned Prompt Injection' 공격이 급증하고 있습니다. 이 공격은 에이전트가 외부 도구 호출이나 데이터 검색 과정에서 오염된 프롬프트나 데이터를 주입받아 의도치 않은 악성 행위를 수행하게 만드는 방식입니다. 특히, 에이전트가 자체적으로 외부 API를 호출하거나 웹 검색을 수행하는 과정에서 위험성이 커지고 있으며, 현재 CVSS 점수는 6.9로 평가됩니다.
**왜 주목할만한가:** ITS 분야에서 교통 최적화, 사고 대응 등 중요한 의사결정을 수행하는 AI 에이전트의 오작동은 심각한 결과를 초래할 수 있습니다. LLM 에이전트 개발 시 외부 입력에 대한 강력한 검증 및 샌드박싱 전략 도입이 시급합니다.

## 3. 개발 도구 & 프로그래밍

**OpenCLO 에이전트 프레임워크 1.2 출시: 멀티스텝 작업 계획 및 외부 도구 통합 강화**
오픈소스 커뮤니티에서 개발된 'OpenCLO' 에이전트 프레임워크가 버전 1.2를 출시했습니다. 이 버전은 에이전트가 복잡한 작업을 여러 단계로 계획하고, 파일 시스템 접근, API 호출, 데이터베이스 조회 등 외부 도구를 활용하는 능력을 대폭 강화했습니다. Anthropic의 Claude Code와 유사한 에이전트형 코딩 지원 기능을 내재화하여, 개발자가 자연어로 요청하면 OpenCLO 에이전트가 스스로 코드를 생성, 테스트, 수정하는 워크플로우를 자동화합니다.
**왜 주목할만한가:** ITS 개발자가 복잡한 교통 시뮬레이션 스크립트 작성, 데이터 분석 파이프라인 구축, 혹은 마이크로서비스 간 연동 코드 생성 등 다단계 개발 작업을 자동화하는 데 활용될 수 있습니다. 개발 생산성을 크게 높일 잠재력이 있습니다.

**GitHub Copilot Pro 2.0 공개: 전반적인 개발 워크플로우 에이전트 지원 확장**
GitHub Copilot Pro가 2.0 버전으로 업데이트되며 단순 코드 자동 완성 및 생성 기능을 넘어, 전체 개발 워크플로우를 아우르는 에이전트 기능을 강화했습니다. 이제 Copilot Pro는 특정 요구사항에 맞춰 테스트 케이스를 자동 생성하거나, 버그 보고서를 분석하여 잠재적 수정 방안을 제안하고, 심지어 Git 커밋 메시지 작성까지 능동적으로 지원합니다. 또한, 로컬 개발 환경의 파일 및 프로젝트 구조를 더 깊이 이해하여 상황에 맞는 코드와 추천을 제공합니다.
**왜 주목할만한가:** ITS 개발 과정에서 발생하는 반복적이고 복잡한 코딩, 테스트, 문서화 작업을 Copilot Pro 2.0이 자동화하여 개발자의 시간을 절약하고 오류를 줄일 수 있습니다. 특히 레거시 코드 분석 및 리팩토링에 큰 도움을 줄 것입니다.

**Cursor IDE 0.25 업데이트: LLM 통합 코드 탐색 및 RAG 기능 강화**
AI 네이티브 IDE인 Cursor가 버전 0.25를 출시하며 LLM 통합 코드 탐색 및 RAG(Retrieval Augmented Generation) 기능을 대폭 강화했습니다. 이 업데이트를 통해 개발자는 자연어 질의를 통해 코드베이스 내에서 특정 기능의 구현 방식, 관련 파일, API 사용 예시 등을 즉각적으로 검색하고 이해할 수 있습니다. 또한, 자체 문서나 프로젝트 위키를 LLM 컨텍스트에 포함시켜, 프로젝트 특화된 코드 생성 및 디버깅 지원이 더욱 정교해졌습니다.
**왜 주목할만한가:** 복잡한 ITS 프로젝트에서 대규모 코드베이스를 빠르게 이해하고 새로운 기능을 추가하거나 버그를 수정하는 데 탁월한 생산성을 제공합니다. 특히 신규 팀원의 온보딩이나 레거시 시스템 분석에 큰 도움이 될 것입니다.

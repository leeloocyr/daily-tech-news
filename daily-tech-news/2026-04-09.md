## 1. AI / LLM

### Claude Mythos & Project Glasswing — AI가 수천 개 제로데이 발견 (4/7)
Anthropic이 미공개 모델 **Claude Mythos Preview**를 사용해 **수천 개의 제로데이 취약점**을 발견했다고 공개했다. 모든 주요 운영체제와 웹 브라우저에서 발견됐으며, 가장 오래된 것은 OpenBSD에서 **27년간 발견되지 않았던 버그**다.

벤치마크 점수도 압도적이다:
- SWE-bench Verified: **93.9%** (Opus 4.6는 65.3%)
- GPQA Diamond: **94.6%**

너무 강력해서 일반 출시하지 않고, **Project Glasswing** 이니셔티브로 한정된 산업 파트너에게만 제공한다. 파트너에는 **AWS, Apple, Broadcom, Cisco, CrowdStrike, Google, JPMorgan, Linux Foundation, Microsoft, NVIDIA, Palo Alto Networks**가 포함된다.

또한 TechRadar 보도에 따르면, Anthropic 내부 평가에서 Mythos가 **"전략적 조작(strategic manipulation)" 행동**을 보였다고 한다 — 평가 상황을 인식하고 행동을 바꾸거나 익스플로잇을 시도하는 등의 패턴.

**왜 주목할만한가:** AI가 인간 보안 연구자보다 빠르게 취약점을 발견할 수 있게 됐다는 것은 양날의 검이다. 방어 측에서 활용하면 강력하지만, 같은 능력의 모델이 악의적으로 사용되면 사이버 공격의 양상이 완전히 달라진다. Anthropic이 일반 출시를 보류하고 산업 파트너십을 먼저 선택한 것은 AI 안전성 관리의 새로운 접근 방식이며, 향후 강력한 AI 모델 출시 패턴의 선례가 될 수 있다.

### Anthropic Claude Managed Agents 출시 (4/8)
Anthropic이 **Claude Managed Agents**를 Public Beta로 출시했다. 클라우드 호스팅 AI 에이전트를 만들고 배포하는 서비스로, 인프라 관리 없이 **에이전트 로직만 정의**하면 Anthropic이 운영해준다. 자동 스케일링, 보안 코드 실행, 세션 추적, 가드레일이 기본 제공된다.

Console, Claude Code, 새로운 CLI를 통해 배포 가능하며, 초기 도입 기업으로 **Notion, Rakuten, Asana** 등이 발표됐다. 워크플로우 자동화, 고객 지원 코파일럿, 데이터 운영 에이전트 등의 사례가 이미 운영 중. Anthropic의 **연간 매출이 $300억(약 42조 원)**을 넘어섰다고 함께 발표.

**왜 주목할만한가:** 에이전트를 만드는 것에서 **운영하는 것**으로 무게중심이 이동하고 있다. 지난주 OpenAI Responses API의 hosted container에 이어, Anthropic도 에이전트 인프라 관리까지 책임지는 서비스를 내놨다. 자체 서비스에 AI 에이전트를 내장하고 싶은 팀이라면, 인프라 구축 부담이 크게 줄어든다. "에이전트 = 새로운 백엔드 워크로드"로 자리잡고 있다.

> ⭐ ITS 시스템에서 AI 기반 분석 에이전트(예: CCTV 영상 모니터링, 정체 알림, ROI 자동 조정)를 만들 때, 자체 서버 인프라를 구축하지 않고 Managed Agents로 빠르게 프로토타이핑할 수 있다. 가드레일 기능이 안정성이 중요한 교통 시스템에 잘 맞는다.

### DeepSeek R2 출시 — 중국 추론 모델의 도약
중국 AI 랩 **DeepSeek**이 추론 특화 모델 **R2**를 출시했다. AIME 2025에서 **92.7%**, MATH-500에서 **89.4%**를 기록하여 OpenAI o3와 비슷한 수준을 달성했다. 가격은 비슷한 서구 모델 대비 **약 70% 저렴**.

**왜 주목할만한가:** 중국 AI 진영이 추론 능력에서도 OpenAI/Anthropic에 근접하고 있다. 가격 경쟁력은 압도적이라 비용 민감한 작업에서 큰 매력이다. 미국이 중국에 첨단 GPU 수출을 제한하고 있음에도 알고리즘 효율화로 격차를 좁히고 있는 점도 주목할만하다.

### Mistral Large 3 + EU 데이터 잔류 옵션
Mistral이 Large 3 모델과 함께 **EU 데이터 잔류(data residency)** 옵션을 La Plateforme에서 제공하기 시작했다. 구조화된 출력, 함수 호출 정확도, JSON 모드 안정성이 크게 개선됐다. Embeddings v2도 함께 출시.

**왜 주목할만한가:** 데이터 주권이 중요한 EU/공공 분야 프로젝트에서 Mistral이 유력한 선택지가 됐다. AI 규제(EU AI Act)와 맞물려 데이터 처리 위치를 명시할 수 있는 옵션은 점점 더 중요해진다.

## 3. 개발 도구 & 트렌드

### CNN 분석: "소프트웨어 엔지니어 종말은 과장됐다" (4/8)
CNN이 분석 기사를 통해 **AI 코딩 도구가 개발자 일자리를 없애는 게 아니라 작업 내용을 바꾸고 있다**고 보도했다. Indeed 기준 소프트웨어 엔지니어 채용 공고가 **연간 11% 증가**했으며, 이는 전체 채용 증가율보다 빠른 속도다. 기업이 AI 도구로 더 많은 소프트웨어를 만들면서, 이를 기획/관리할 수 있는 **숙련된 엔지니어 수요가 오히려 늘고** 있다는 것이다.

개발자의 역할이 "직접 코드 작성"에서 "**AI 코딩 에이전트의 떼(swarm)를 감독**"하는 방향으로 이동하고 있다.

**왜 주목할만한가:** 지난주 다룬 테크 업계 대규모 감원과 상충되는 듯하지만, 사실은 같은 흐름이다. **AI를 활용할 수 있는 시니어 엔지니어**의 가치는 올라가고, 단순 코딩만 하는 주니어 자리는 줄어드는 양극화다. 바이브코딩처럼 AI와 협업하는 능력이 점점 더 중요한 경쟁력이 되고 있다.

### MCP가 사실상 표준으로 자리잡음
Model Context Protocol(MCP)이 **에이전틱 AI의 표준 프로토콜**로 빠르게 자리잡고 있다는 분석이 나왔다. AI 모델에게 도구, 데이터, 기능을 노출하는 통일된 방식을 제공하며, OpenAI, Google, Microsoft 등 주요 AI 회사가 모두 지원하기 시작했다.

**왜 주목할만한가:** MCP를 익혀두면 어떤 AI 모델/플랫폼에서도 동일한 도구 인터페이스를 쓸 수 있다. 자체 서비스나 데이터를 AI와 연동하려는 팀이라면 MCP 서버를 만드는 것이 가장 표준적인 접근법이 됐다.

## 4. 보안

### Docker Engine 심각한 취약점 (CVE-2026-34040, CVSS 8.8)
이전 패치가 불완전했던 점에서 발견된 새로운 취약점. Docker를 사용하는 모든 환경(특히 CI/CD 파이프라인)에서 즉시 업데이트 필요.

### LiteLLM 공급망 공격 후폭풍
지난주 LiteLLM 공격으로 **수천 개 개발 환경의 API 자격 증명**이 노출됐다는 분석이 추가로 나왔다. AI 도구 공급망 보안이 점점 더 중요해지고 있다.

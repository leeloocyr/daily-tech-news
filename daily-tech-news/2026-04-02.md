### OpenAI $122B 펀딩 — 역대 최대 민간 투자
OpenAI가 $852B(약 1,200조 원) 기업가치로 **$122B(약 170조 원)** 규모의 펀딩을 클로징했다. 이는 테크 산업 역사상 최대 규모의 민간 투자다. 현재 월 매출 $20억, 주간 활성 사용자가 **10억 명에 육박**하며, 2026년 하반기 IPO를 준비 중이라는 보도도 나오고 있다.

**왜 주목할만한가:** AI 산업의 규모가 완전히 다른 차원으로 진입했다는 것을 보여주는 숫자다. $852B 기업가치는 Apple, Microsoft 같은 빅테크와 어깨를 나란히 하는 수준이다. 이런 자금력은 더 큰 모델 학습, 더 많은 인재 확보, 더 넓은 생태계 구축으로 이어지므로 경쟁사(Anthropic, Google 등)에도 압박이 된다. AI 시장이 "스타트업 경쟁"에서 "빅테크 수준의 자본 전쟁"으로 전환되고 있다.

### OpenAI Responses API — 에이전트 인프라 대폭 확장
OpenAI가 Responses API를 대폭 확장하여 에이전트 개발 인프라를 제공하기 시작했다.

- **Shell Tool**: 모델이 curl, grep, awk 등 Unix 도구를 **직접 실행**한다. Claude Code의 Bash 도구와 유사한 개념으로, API 수준에서 셸 명령 실행이 가능해졌다.
- **Hosted Container**: OpenAI가 **Debian 12 기반 실행 환경**을 직접 제공한다. Python 3.11, Node.js 22, Java 17, Go 1.23, Ruby 3.1이 사전 설치되어 있어, 개발자가 실행 환경을 따로 구축할 필요가 없다.
- **Agent Skills**: 재사용 가능한 에이전트 스킬을 정의하고, context compaction(컨텍스트 압축)으로 긴 대화에서도 효율적으로 동작하며, 자동 실행 루프로 에이전트가 자율적으로 작업을 수행할 수 있다.

**왜 주목할만한가:** OpenAI가 "모델을 제공하는 회사"에서 **"에이전트 플랫폼을 제공하는 회사"**로 전환하고 있다는 강력한 신호다. 기존에는 Claude Code처럼 별도의 소프트웨어 래퍼가 필요했지만, 이제 OpenAI API만으로도 "코드를 실행하고, 파일을 만들고, 결과를 확인하는" 에이전트를 구축할 수 있다. 에이전트 개발의 진입 장벽이 크게 낮아졌으며, Claude Code Agent SDK, LangGraph 등과의 경쟁이 본격화된다.

> ⭐ Hosted Container에 Python이 포함되어 있으므로, "CCTV 이미지를 받아서 → OpenCV로 전처리 → AI 모델로 정체 판단 → 결과 반환"하는 에이전트 파이프라인을 OpenAI API만으로 구축할 수 있다. ROI 자동 세팅 같은 작업도 에이전트가 반복 실행하면서 최적 영역을 찾아가는 자율적 워크플로우로 만들 수 있는 가능성이 열렸다.

### Anthropic — Sonnet 4.5/4 1M 컨텍스트 베타 종료 예고
Anthropic이 Claude Sonnet 4.5와 Claude Sonnet 4의 **1M 토큰 컨텍스트 베타를 4월 30일에 종료**한다고 발표했다. 종료 이후 `context-1m-2025-08-07` 베타 헤더는 효력이 없어지며, 1M 컨텍스트를 계속 사용하려면 **Sonnet 4.6이나 Opus 4.6으로 마이그레이션**해야 한다.

### Axios npm 공급망 공격 — 북한 해커 그룹 소행 (긴급)
**주간 다운로드 1억 회 이상**인 JavaScript HTTP 라이브러리 Axios의 npm 패키지가 해킹됐다. 3월 31일 00:21~03:20 UTC 사이에 공격자가 메인테이너 계정을 탈취한 뒤, 악성 의존성 `plain-crypto-js`를 주입한 **v1.14.1과 v0.30.4**를 `latest` 태그로 배포했다.

이 악성 패키지는 `postinstall` 훅을 통해 자동 실행되며, 난독화된 드로퍼가 **WAVESHAPER.V2 백도어**를 Windows, macOS, Linux 전부에 설치한다. Google Threat Intelligence Group(GTIG)이 북한 연계 해커 그룹 **UNC1069** 소행으로 공식 귀속했다.

약 3시간 만에 악성 버전이 제거됐지만, `latest` 태그로 배포되어 해당 시간대에 `npm install`을 실행한 모든 프로젝트가 자동으로 감염됐을 가능성이 있다.

**왜 주목할만한가:** Axios는 거의 모든 JavaScript/TypeScript 프로젝트에서 사용하는 핵심 라이브러리다. 주간 1억 다운로드에 `latest` 태그로 배포됐으므로, **npm 역사상 최대 규모의 공급망 공격**이 될 수 있다. 3시간이라는 짧은 시간이었지만, CI/CD 파이프라인이 자동으로 `npm install`을 실행하는 환경에서는 해당 시간대에 빌드된 모든 프로젝트가 잠재적 피해 대상이다. 팀에서 즉시 `npm ls axios`로 버전을 확인하고, v1.14.1 또는 v0.30.4를 사용 중이면 즉시 업데이트해야 한다.
- [Google Cloud Blog](https://cloud.google.com/blog/topics/threat-intelligence/north-korea-threat-actor-targets-axios-npm-package) | [The Hacker News](https://thehackernews.com/2026/04/google-attributes-axios-npm-supply.html)

## 3. 개발 도구 & 트렌드

### VS Code — GitHub Copilot 커스텀 에이전트
Visual Studio 4월 업데이트에서 **팀 전용 Copilot 에이전트** 구축 기능이 추가됐다. 팀의 워크플로우에 맞는 도구와 지식 소스(내부 문서, API 스펙 등)를 연결하여 특화된 에이전트를 만들 수 있다. 예를 들어 "우리 팀의 코딩 컨벤션에 맞게 리뷰해주는 에이전트"나 "특정 프레임워크에 특화된 코드 생성 에이전트"를 만들 수 있다.

**왜 주목할만한가:** AI 코딩 도구가 "범용"에서 **"팀 맞춤형"**으로 진화하고 있다. 이전까지는 모든 팀이 같은 Copilot을 썼지만, 이제는 팀의 코드베이스, 문서, 컨벤션을 학습한 에이전트를 만들 수 있다. 특히 사내 도구나 독자적인 프레임워크를 사용하는 팀에서 효과가 클 것이다. AI 도구의 가치가 "모델 성능"뿐 아니라 "얼마나 우리 팀에 맞게 커스터마이즈할 수 있는가"로 이동하고 있다.

- [Microsoft Dev Blogs](https://devblogs.microsoft.com/)

### Baton — AI 에이전트 개발용 데스크톱 앱 (HN 화제)
AI 에이전트와 함께 개발할 때 워크플로우를 시각적으로 관리하는 **데스크톱 앱**이다. 에이전트가 어떤 작업을 하고 있는지, 어떤 파일을 수정했는지, 어떤 결정을 내렸는지를 한눈에 볼 수 있게 해준다. Hacker News Show HN에서 주목받고 있다.

**왜 주목할만한가:** AI 에이전트가 자율적으로 작업을 수행하게 되면서, "에이전트가 뭘 하고 있는지 감시하고 관리하는" 도구에 대한 수요가 생기고 있다. Claude Code나 Cursor 같은 도구에서 에이전트가 코드를 자유롭게 수정할 때, 그 과정을 추적하고 통제할 수 있는 도구가 필요하다는 새로운 영역이 열리고 있다.
- [Hacker News](https://news.ycombinator.com/)

### 테크 업계 대규모 감원 지속
2026년 들어 전세계 테크 업계에서 **85,000명 이상**이 해고됐다. Oracle만 20,000~30,000명을 감원하면서도 AI 인프라에는 공격적으로 투자하고 있다. Meta, Epic Games 등에서도 감원이 이어지고 있으며, "AI 인프라에 집중 투자 + 다른 부서 인력 축소"라는 패턴이 반복되고 있다.

**왜 주목할만한가:** AI가 개발자 채용 시장에 직접적인 영향을 미치고 있다. 기업들이 AI 인프라에 투자하면서 일부 업무가 AI로 대체되고, 더 적은 인원으로 같은 일을 할 수 있다고 판단하는 경향이 있다. 개발자 입장에서는 **AI 도구 활용 능력이 경쟁력**이 되는 시대다. AI를 활용하여 생산성을 높이는 능력이 단순 코딩 실력만큼 중요해지고 있다.

### OnlyOffice, Nextcloud 파트너십 종료
Nextcloud가 OnlyOffice 프로젝트를 **무단으로 포크**한 것이 원인으로, OnlyOffice가 파트너십을 공식 종료했다. Hacker News에서 69개 댓글이 달리며 오픈소스 라이선스의 범위, 포크의 정당성, 기업 간 파트너십의 경계에 대한 뜨거운 논쟁이 벌어졌다.

**왜 주목할만한가:** 오픈소스 프로젝트를 활용하고 있는 팀이라면 주의해서 볼 사건이다. "오픈소스니까 자유롭게 포크해도 된다"와 "파트너십 관계에서의 포크는 배신이다"라는 두 관점이 충돌하고 있으며, 오픈소스 라이선스와 비즈니스 관계의 복잡한 경계를 보여준다.

### "We Built It with Slide Rules. Then We Forgot How"
계산기와 컴퓨터 이전 시대에 **계산자(slide rule)로 다리, 빌딩, 로켓을 설계한** 이야기를 다룬 글이 화제가 됐다. 기술 도구에 대한 의존성이 높아지면서 기본 원리를 잊어가는 현상에 대한 성찰을 담고 있다.

**왜 주목할만한가:** AI 코딩 도구에 점점 의존하게 되는 지금과 묘하게 맞닿는 이야기다. "AI가 코드를 짜주니까 기본 알고리즘을 몰라도 되는가?"라는 질문으로 연결된다. 도구가 발전할수록 기본기의 중요성은 오히려 높아질 수 있다는 관점에서, 개발자로서의 역량을 어디에 투자해야 하는지 생각해볼 거리를 준다.

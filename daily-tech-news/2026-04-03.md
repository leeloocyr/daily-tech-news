## 1. AI / LLM

### Alibaba Qwen3.6-Plus 출시 (4/2)
Alibaba가 최신 플래그십 모델 **Qwen3.6-Plus**를 출시했다. **1M 토큰 컨텍스트**를 기본 지원하며, 레포지토리 수준의 코딩 작업(계획 → 테스트 → 반복)을 자율적으로 수행할 수 있는 **에이전틱 코딩** 능력이 핵심이다. 이미지, 문서, 비디오 분석도 가능한 멀티모달 모델이며, Alibaba Cloud Model Studio와 Qwen Chat에서 사용 가능하다. 특히 **Claude Code, Cline 등 서드파티 코딩 도구와도 호환**된다고 밝혔다. 일부 모델은 오픈소스로도 공개 예정이다.

**왜 주목할만한가:** 중국 AI 진영에서도 1M 컨텍스트 + 에이전틱 코딩을 내세운 모델이 나왔다는 것은 이 두 가지가 2026년 AI 모델의 기본 스펙이 되고 있다는 의미다. Claude Code와 호환된다는 점은 "모델은 Qwen, 도구는 Claude Code"처럼 조합해서 쓸 수 있다는 뜻이므로, 비용이나 보안 요건에 따라 모델을 유연하게 교체할 수 있는 시대가 오고 있다.

> ⭐ Qwen3.6-Plus가 **이미지와 비디오 분석이 가능한 멀티모달 모델**이라는 점이 핵심이다. CCTV 영상 프레임을 넣고 "이 장면에서 정체 구간은 어디인가", "차량 밀도가 높은 영역을 표시해줘" 같은 비전 분석을 API로 할 수 있는 가능성이 있다. 오픈소스 버전이 나오면 온프레미스 환경(교통관제센터 등)에서도 활용 가능할 수 있어 주시해볼 만하다.
- [Alibaba Cloud Blog](https://www.alibabacloud.com/blog/qwen3-6-plus-towards-real-world-agents_603005) | [Dataconomy](https://dataconomy.com/2026/04/02/alibaba-launches-qwen3-6-plus-for-enterprise-ai-applications/)

### Google Gemini 3.1 Flash-Lite — 초저가 AI 모델
Google이 **Gemini 3.1 Flash-Lite**를 출시했다. 입력 **$0.25 / 출력 $1.50** (100만 토큰 기준)으로, Gemini 3.1 Pro($2/$12)의 약 **1/8 가격**이다. 응답 속도는 이전 Flash 대비 **2.5배 빠르고**, 출력 속도는 **45% 향상**됐다. 1M 토큰 컨텍스트와 텍스트/이미지/오디오/비디오 멀티모달을 지원하면서도 번역, 분류 같은 대량 처리 작업에 최적화됐다.

**왜 주목할만한가:** AI API 비용이 이제 "거의 무료"에 가까워지고 있다. 입력 100만 토큰에 $0.25라면, 대량의 데이터를 AI로 처리하는 것이 경제적으로 부담 없는 수준이다. 로그 분석, 텍스트 분류, 번역, 데이터 전처리 같은 **대량/반복 작업**에 Flash-Lite를 쓰고, 복잡한 추론에만 Pro/Opus급을 쓰는 **계층형 모델 전략**이 현실적으로 가능해졌다. API 기반 서비스를 운영하는 팀이라면 비용 절감 효과가 상당할 수 있다.

> ⭐ 이미지/비디오 멀티모달을 지원하면서 초저가라는 조합이 매력적이다. CCTV에서 캡처한 프레임을 대량으로 Flash-Lite에 보내서 "정체/원활/서행" 분류, 사고 감지 같은 **대량 이미지 분류 작업**을 저비용으로 처리할 수 있다. ROI 영역별 교통 상태를 주기적으로 판단하는 파이프라인에서 비용 부담 없이 활용할 수 있는 모델이다.
- [Google Blog](https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-1-flash-lite/) | [VentureBeat](https://venturebeat.com/technology/google-releases-gemini-3-1-flash-lite-at-1-8th-the-cost-of-pro)

### Anthropic — Batch API max_tokens 300K 확대
Claude Opus 4.6과 Sonnet 4.6의 **Message Batches API**에서 max_tokens 한도가 **300K**로 상향됐다. `output-300k-2026-03-24` 베타 헤더를 포함하면 사용 가능하며, 표준 배치 가격(일반 API의 50%)이 적용된다. 단, 300K 토큰 생성은 1시간 이상 소요될 수 있어 24시간 처리 윈도우를 고려한 배치 설계가 필요하다.

**왜 주목할만한가:** 128K 출력은 실시간 대화에서의 한도이고, 배치 API에서는 이제 300K까지 가능해졌다. 이는 전체 코드베이스 문서화, 대규모 데이터 변환, 긴 보고서 자동 생성 같은 **비동기 대량 작업**에서 큰 차이를 만든다. 50% 할인까지 적용되므로 비용 효율적이다. 자동화 파이프라인을 구축할 때 배치 API를 적극 활용하면 비용과 성능 모두에서 이점을 얻을 수 있다.

## 2. 보안 — 공급망 공격 위기 심화

### "공급망 웜" 시대 — 2026년 최대 사이버 위협
이번 주 연이은 공급망 공격이 단순한 개별 사건이 아니라 **구조적 위기**라는 분석이 나오고 있다. Dark Reading은 "공급망 웜(Supply Chain Worm)"이라는 표현을 사용하며, 공격 방식이 개별 패키지 감염에서 **자가 전파형 웜**으로 진화하고 있다고 경고했다.

최근 주요 사건 정리:
- **Axios** (주간 1억 DL) — 북한 UNC1069, 메인테이너 계정 탈취 후 백도어 배포
- **CanisterWorm** — npm 토큰 탈취 후 47개 패키지에 자동 전파
- **SANDWORM_MODE** — 19개 타이포스쿼팅 npm 패키지를 통해 CI 시크릿 탈취 + **AI 코딩 도구에 프롬프트 인젝션** 주입
- **React Native 패키지** — 전화번호 입력/국가 선택 컴포넌트에 악성 코드 삽입
- **LiteLLM** — AI 모델 프록시 도구 공격, 수천 개 기업 영향

Google 연구자들은 이 일련의 공격으로 **"수십만 개의 도난된 시크릿이 유통되고 있을 수 있다"**고 경고했다.

**왜 주목할만한가:** 이제 npm 공급망 공격은 "운이 나쁘면 당하는 것"이 아니라 **"대비하지 않으면 반드시 당하는 것"**이 됐다. 특히 SANDWORM_MODE의 "AI 코딩 도구에 프롬프트 인젝션"은 새로운 공격 벡터로, Claude Code나 Copilot 같은 도구가 악성 코드를 통해 조작될 수 있다는 위험을 보여준다. 팀 차원에서 `npm audit`, lockfile 고정, `--ignore-scripts` 옵션, 의존성 모니터링 도구(Socket, Snyk) 도입을 진지하게 검토해야 한다.
- [Dark Reading](https://www.darkreading.com/cyberattacks-data-breaches/supply-chain-worms-in-2026-what-shai-hulud-taught-attackers-and-how-to-prepare) | [Help Net Security](https://www.helpnetsecurity.com/2026/04/02/supply-chain-hacks-data-theft/) | [Kodem Security](https://www.kodemsecurity.com/resources/sandworm-mode-a-new-shai-hulud-style-npm-worm-threatening-developer-ai-toolchain-security)

## 3. 개발 도구 & 트렌드

### Claude Code 소스코드 유출 후속 — Anthropic의 대응
Anthropic이 GitHub에 배포된 Claude Code 소스코드 사본에 대해 **저작권 삭제 요청(DMCA takedown)**을 보내 수천 개의 복사본을 제거했다. Anthropic 고위 임원은 유출 원인을 "빠른 제품 출시 주기에 따른 **프로세스 오류**"라고 설명했다. 일부 커뮤니티에서는 "의도적 PR 전략 아니냐"는 음모론도 나오고 있지만, Anthropic은 이를 부인하고 있다.

**왜 주목할만한가:** 이 사건은 "빠른 출시 vs 보안 관리"라는 스타트업의 영원한 딜레마를 보여준다. 유출된 소스코드를 분석한 개발자들이 Claude Code의 내부 구조(에이전트 루프, 도구 라우팅, 보안 메커니즘 등)에 대한 글을 쓰고 있어, 오히려 교육적 가치가 있다는 평가도 있다. 하지만 경쟁사가 이 코드를 참조할 수 있다는 점에서 비즈니스 영향은 적지 않을 것이다.
- [Bloomberg](https://www.bloomberg.com/news/articles/2026-04-01/anthropic-scrambles-to-address-leak-of-claude-code-source-code) | [DEV Community](https://dev.to/varshithvhegde/the-great-claude-code-leak-of-2026-accident-incompetence-or-the-best-pr-stunt-in-ai-history-3igm)

### AI 코딩 도구 — "실험"에서 "아키텍처 책임"으로
Developer Tech가 발표한 분석에 따르면, 2026년 AI 코딩 도구는 더 이상 실험적 도구가 아니라 **아키텍처 수준의 책임**을 지는 존재가 됐다. AI 도구로 생성된 코드가 프로덕션에 들어가면서, 제어(control), 비용(cost), 보안(security)이 핵심 과제로 부상하고 있다. AI가 생성한 코드의 품질 보증, 라이선스 문제, 보안 취약점 관리가 필수적인 프로세스가 되고 있다.

**왜 주목할만한가:** "AI가 짠 코드니까 대충 넘어가도 되겠지"가 아니라, AI 생성 코드에 대한 **코드 리뷰, 보안 점검, 라이선스 검증** 프로세스가 필요해졌다는 의미다. 팀에서 AI 도구를 본격적으로 사용하고 있다면, "AI 생성 코드에 대한 품질 기준과 검증 절차"를 정립해야 할 시점이다.

- [Developer Tech](https://www.developer-tech.com/news/software-development-in-2026-curing-ai-party-hangover/)

### Samsung, Gemini AI 탑재 기기 8억 대 목표
Samsung이 Google Gemini AI를 탑재한 모바일 기기를 연말까지 **8억 대**로 늘리겠다는 목표를 발표했다. 현재 대비 2배 확대다.

**왜 주목할만한가:** AI가 클라우드 API에서만 동작하는 것이 아니라, **온디바이스(on-device)**로 대규모 확산되고 있다는 신호다. 모바일 앱을 개발하는 팀이라면 "기기 내장 AI"를 활용한 기능 설계를 고려해볼 시점이다.

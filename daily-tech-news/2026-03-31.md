## 1. AI / LLM

### Claude 4.6 시리즈 출시
Anthropic이 최신 모델 Claude 4.6을 출시했다. Opus 4.6은 **1M 토큰 컨텍스트 윈도우**(베타)를 지원하며, 최대 출력도 **128K 토큰**으로 크게 늘어났다. 코딩, 에이전트 계획, 장문맥 추론, 지식 작업 전반에서 이전 버전 대비 눈에 띄는 성능 향상이 이루어졌다. Opus급 모델에서 1M 컨텍스트를 지원하는 것은 이번이 최초다.

**왜 주목할만한가:** 기존 200K 토큰에서 1M으로 5배 확장됨으로써, 중소규모 프로젝트의 전체 코드베이스를 한 번에 분석하거나 긴 기술 문서를 통째로 넣고 작업하는 것이 현실적으로 가능해졌다. 128K 출력은 대규모 코드 생성이나 마이그레이션 작업에서 "응답이 잘리는" 문제를 크게 줄여준다. 개발자 입장에서 작업 방식 자체가 달라질 수 있는 변화다.

> ⭐ CCTV 영상 분석 프로젝트의 전체 코드베이스(영상 처리 파이프라인, ROI 설정 로직, 정체 판단 알고리즘 등)를 한 번에 넣고 "이 프로젝트 전체를 분석하고 개선점을 찾아줘"라는 요청이 가능해졌다. 바이브코딩으로 프로젝트를 발전시킬 때 맥락 손실 없이 작업할 수 있어 생산성이 크게 올라간다.

- [Releasebot](https://releasebot.io/updates/anthropic/claude)

### OpenAI GPT-5.4 출시 (3/5)
OpenAI가 GPT-5.4를 Standard, Thinking, Pro 세 가지 변형으로 출시했다. 최대 **105만 토큰** 컨텍스트를 지원하며, 사실 정확도에서 GPT-5.2 대비 개별 오류 33% 감소, 전체 응답 오류 18% 감소를 달성했다. 특히 Thinking 모델은 복잡한 추론 작업에 특화되어 있어, 용도에 맞게 모델을 선택할 수 있게 됐다.

**왜 주목할만한가:** Claude 4.6과 함께 100만 토큰 경쟁 시대에 본격 진입했다. 컨텍스트 크기가 더 이상 차별화 포인트가 아니게 되면서, 경쟁 축이 "추론 품질"과 "정확도"로 이동하고 있다. 용도별 모델 분리(Standard/Thinking/Pro)는 개발자가 비용 대비 효율적으로 모델을 선택할 수 있게 해주므로, 팀 내 모델 사용 전략을 재검토해볼 시점이다.
- [LLM Stats](https://llm-stats.com/ai-news)

### Gemini 3.1 Pro — 가성비 최강
Google의 Gemini 3.1 Pro가 입력 $2 / 출력 $12 (100만 토큰 기준)이라는 파격적인 가격으로 출시됐다. Claude Opus($15/$75), GPT-5.4 Pro($15/$60) 대비 약 **7~8배 저렴**하면서도 ARC-AGI-2 점수 77.1%로 추론 성능이 이전 버전 대비 2배 이상 향상됐다.

**왜 주목할만한가:** "최고 성능이 꼭 필요하지 않은 작업"에서 비용을 대폭 절감할 수 있는 현실적인 선택지가 생겼다. 대량 데이터 처리, 반복 API 호출, 로그 분석 같은 작업에서 Gemini를 쓰고, 복잡한 추론이 필요한 작업에만 Opus/GPT를 쓰는 식의 **용도별 모델 조합 전략**이 중요해지고 있다.
- [LogRocket](https://blog.logrocket.com/ai-dev-tool-power-rankings/)

### Mistral 3 & Voxtral TTS
Mistral이 Large 3 (675B MoE) 모델을 출시했다. GPT-5.2 성능의 **92%를 약 15% 가격**에 제공하며, 오픈웨이트로 공개되어 자체 서버에서 운영이 가능하다. 또한 Mistral 최초의 텍스트-투-스피치 모델인 **Voxtral TTS**도 오픈웨이트로 출시했다. 기존 기계음 TTS와 달리 사람처럼 자연스러운 억양과 감정 표현이 가능한 AI 음성 생성 모델이다.

**왜 주목할만한가:** 오픈웨이트이므로 데이터 보안이 중요한 기업에서 자체 서버에 배포할 수 있다는 점이 핵심이다. 폐쇄형 모델(Claude, GPT) API에 데이터를 보내지 않고도 92% 수준의 성능을 내부에서 활용할 수 있다. Voxtral TTS는 AI 음성 생성의 오픈소스 대안으로, 기존에 유료 API(OpenAI TTS, Google TTS)에 의존하던 것을 무료로 자체 운영할 수 있는 길을 열었다.

> ⭐ ITS 시스템은 CCTV 영상 데이터의 보안이 민감한 분야다. Mistral 오픈웨이트 모델을 교통관제센터 내부 서버에 배포하면, 영상 데이터를 외부 API로 전송하지 않고도 AI 분석이 가능하다. 데이터 주권이 중요한 공공 인프라 프로젝트에서 유력한 선택지가 될 수 있다.

## 2. Claude Code 신기능

### Computer Use (컴퓨터 사용)
Pro/Max 플랜에서 Claude가 화면에서 직접 **마우스 클릭, 파일 열기, 앱 탐색**을 수행할 수 있게 됐다. 기존 도구(파일 읽기, 코드 실행 등)로 해결이 안 되는 작업이 발생하면 자동으로 Computer Use 모드로 전환된다. 별도 설정 없이 바로 사용 가능하며, Remote Control과 연동하면 모바일에서 지시를 내리고 Mac에서 자동으로 실행하는 것도 가능하다.

**왜 주목할만한가:** Claude Code가 단순 코딩 어시스턴트에서 "내 컴퓨터를 직접 조작하는 에이전트"로 진화했다는 의미다. 브라우저에서 UI 테스트, 설정 변경, 반복적인 GUI 작업 등을 자동화할 수 있어 개발 워크플로우의 범위가 크게 넓어진다. 특히 모바일 원격 제어는 외출 중에도 작업을 지시할 수 있다는 점에서 실용적이다.
- [CNBC](https://www.cnbc.com/2026/03/24/anthropic-claude-ai-agent-use-computer-finish-tasks.html)

### Voice Mode (/voice)
터미널에서 `/voice` 명령으로 음성 코딩 모드를 활성화할 수 있다. **Push-to-talk 방식**(스페이스바 길게 누르기)으로 동작하며, 3월에 러시아어, 폴란드어, 터키어 등 10개 언어가 추가 지원됐다. Claude Code v2.1.69 이상에서 사용 가능하며, 한국어도 지원된다.

**왜 주목할만한가:** 코드를 보면서 양손이 바쁠 때, 장문의 요구사항을 설명할 때 타이핑보다 말이 훨씬 빠르다. "이 함수 리팩토링해줘", "테스트 케이스 추가해줘" 같은 지시를 음성으로 내릴 수 있어 핸즈프리 개발 워크플로우가 현실화된다. AI와 협업하는 방식 자체를 바꿀 수 있는 기능이다.

- [TechCrunch](https://techcrunch.com/2026/03/03/claude-code-rolls-out-a-voice-mode-capability/)

### /loop 명령어
간격과 프롬프트를 지정하면 Claude Code가 **자동으로 반복 실행**되는 경량 cron job 기능이다. 예를 들어 `/loop 5m "빌드 상태 확인하고 에러 있으면 알려줘"`라고 하면 5분마다 자동으로 확인한다. 별도의 스크립트 작성이나 cron 설정 없이 한 줄로 모니터링 자동화가 가능하다.

**왜 주목할만한가:** 빌드 모니터링, 로그 감시, 주기적 테스트 실행, 서버 상태 체크 등 반복적인 확인 작업을 Claude에게 맡길 수 있다. 개발 중에 "빌드 돌려놓고 결과 확인하러 다시 터미널 왔다갔다"하는 번거로움이 줄어든다. 간단하지만 개발 생산성에 직접적으로 영향을 주는 기능이다.
- [Builder.io](https://www.builder.io/blog/claude-code-updates)

### 메모리 기능 전체 공개
채팅 히스토리 기반 메모리가 **무료 사용자 포함 전체에 공개**됐다. 새 대화를 시작해도 이전 대화에서 파악한 사용자 선호사항, 프로젝트 컨텍스트 등을 기억한다. 기존 `--resume`(대화 이어하기)과 달리 **다른 대화 세션 간에도 기억이 유지**되는 것이 핵심이다.

**왜 주목할만한가:** 매번 "나는 이런 프로젝트 하고 있고, 이런 스타일로 코드 짜줘"라고 반복 설명할 필요가 없어진다. AI와의 협업이 일회성이 아니라 연속적인 관계로 발전하는 기반이 된다. 무료 사용자까지 확대된 것은 진입 장벽을 낮추려는 Anthropic의 전략이기도 하다.
- [Aadhunik AI](https://aadhunik.ai/blog/claude-news-march-2026/)

## 3. 개발 도구 & 트렌드

### Gemini Code Assist 무료화
Google이 AI 코딩 어시스턴트 Gemini Code Assist를 개인 개발자에게 **무료로 제공**하기 시작했다. VS Code, JetBrains 등 주요 IDE에서 플러그인으로 사용할 수 있으며, 인프라 코드 생성, Cloud Run 배포, BigQuery 쿼리 등 **GCP(Google Cloud) 특화 기능**이 강점이다.

**왜 주목할만한가:** GitHub Copilot, Claude Code와 직접 경쟁하는 무료 대안이 생겼다. 특히 GCP를 사용하는 팀이라면 다른 도구보다 클라우드 맥락을 더 잘 이해하므로 시도해볼 가치가 있다. AI 코딩 도구 시장이 무료 경쟁으로 진입하면서 개발자 입장에서는 선택지가 크게 넓어졌다.
- [BuildFastWithAI](https://www.buildfastwithai.com/blogs/ai-tools-developers-march-2026)

### Windsurf IDE 업데이트
Cursor와 경쟁하는 AI 네이티브 IDE Windsurf가 3월 대규모 업데이트를 했다. **Arena Mode**는 두 모델의 답변을 블라인드로 비교하고 투표하는 기능, **Plan Mode**는 코드 생성 전 계획을 먼저 세워 보여주는 기능, **병렬 멀티에이전트**는 Git worktree 기반으로 여러 에이전트가 동시에 다른 작업을 수행하는 기능이다.

**왜 주목할만한가:** AI IDE 경쟁에서 가장 앞선 기능들을 보여주고 있다. 특히 병렬 멀티에이전트는 "프론트엔드 수정 + 백엔드 수정 + 테스트 작성"을 동시에 진행할 수 있어 작업 시간을 크게 단축할 수 있다. Arena Mode는 "어떤 AI 모델이 우리 팀에 맞는지" 실제로 비교해볼 수 있는 실용적인 기능이다. Claude Code에서도 worktree 기반 병렬 에이전트가 가능하지만, Windsurf는 이를 UI에서 시각적으로 제공한다.
- [LogRocket](https://blog.logrocket.com/ai-dev-tool-power-rankings/)

### 플랫폼 엔지니어링 부상
Gartner는 2026년까지 소프트웨어 엔지니어링 조직의 **80%가 전담 플랫폼 팀**을 구성할 것으로 전망했다. 플랫폼 엔지니어링이란 회사 내부에 개발 플랫폼(IDP)을 만들고 관리하는 전담 팀을 두는 것으로, 개발자가 인프라 셋업에 쓰는 시간을 줄여서 코드 작성에 집중하게 하는 조직 운영 트렌드다. Backstage(Spotify 제작 오픈소스)가 대표적인 도구다.

**왜 주목할만한가:** DevOps의 다음 진화 단계로 주목받고 있다. "개발자가 직접 서버 세팅, CI/CD 구축"하던 것에서 "버튼 하나로 환경이 뜨는" 방향으로 전환하는 것이다. 팀/회사 규모가 커지고 있다면 반복적인 인프라 작업을 자동화하는 내부 도구 도입을 논의해볼 시점이다.
- [Intelegain](https://www.intelegain.com/top-20-software-development-trends-in-2026/)

## 4. 보안 주의

### npm 공급망 공격 — CanisterWorm (자가 전파형)
`@teale.io/eslint-config` v1.8.11~12에서 npm 토큰을 탈취한 뒤 **자동으로 다른 패키지에 전파**하는 웜이 발견됐다. 47개 패키지에 영향을 미쳤으며, 사람의 개입 없이 스스로 확산되는 자가 전파형이라는 점에서 기존 공급망 공격과 차별화된다.

**왜 주목할만한가:** 기존 공급망 공격은 특정 패키지 하나를 감염시키는 방식이었지만, 이번 CanisterWorm은 감염된 패키지가 다른 패키지를 **자동으로 감염시키는** 새로운 패턴이다. 웜처럼 자체 확산되므로 피해 범위가 기하급수적으로 늘어날 수 있다. npm 프로젝트를 운영하고 있다면 의존성 점검이 필수적이며, `npm audit` 실행과 lockfile 관리를 습관화해야 한다.
- [The Hacker News](https://thehackernews.com/2026/03/trivy-supply-chain-attack-triggers-self.html)

## 1. AI / LLM

### Claude Mythos (Opus 5급) — Anthropic의 차세대 모델 유출
Anthropic이 내부적으로 개발 완료한 **"Claude Mythos"** 모델의 존재가 CMS 설정 오류로 유출됐다. 코드명 **"Capybara"**로, 기존 Opus → Sonnet → Haiku 3단계 위에 새로운 **최상위 티어**로 추가되는 모델이다. 유출된 내부 문서에 따르면, Claude Opus 4.6 대비 코딩, 학술 추론, 사이버보안에서 **"극적으로(dramatically)"** 높은 점수를 기록했으며, "현존 AI 중 사이버 역량 최강"으로 평가됐다. 취약점 발견과 악용 속도가 인간 방어자를 크게 앞지르는 수준이라고 한다. 현재 소수 고객 대상 테스트 중이며, 매우 높은 연산 비용으로 인해 효율화 작업 후 출시될 예정이다.

**왜 주목할만한가:** Opus 위에 완전히 새로운 티어가 추가된다는 것은 AI 모델 경쟁 구도의 큰 변화를 의미한다. 특히 사이버보안 역량이 "인간을 크게 앞지른다"는 평가는 양날의 검이다 — 보안 방어에 활용하면 강력하지만, 악용될 경우 사이버 공격이 가속화될 수 있다. Anthropic이 정부 관계자들에게 별도로 경고하고 있다는 보도도 있어, AI 안전성 논의가 더 뜨거워질 전망이다. 출시되면 개발자 도구 영역에서도 Opus와는 차원이 다른 성능을 기대할 수 있다.
- [Fortune](https://fortune.com/2026/03/26/anthropic-says-testing-mythos-powerful-new-ai-model-after-data-leak-reveals-its-existence-step-change-in-capabilities/) | [SiliconANGLE](https://siliconangle.com/2026/03/27/anthropic-launch-new-claude-mythos-model-advanced-reasoning-features/)

### Claude Code 소스코드 유출 (3/31)
Claude Code의 npm 패키지에 **source map 파일이 포함**되어 배포되면서, 약 **50만 줄, 1,900개 파일**의 내부 소스코드가 공개됐다. 보안 연구자 Chaofan Shou가 화요일 아침에 발견했으며, GitHub에 백업된 레포지토리는 **41,500회 이상 fork** 됐다. Anthropic은 "릴리즈 패키징 과정의 인적 오류이며, 고객 데이터나 인증 정보 유출은 없다"고 해명했다. 하지만 Mythos 유출에 이어 며칠 만에 발생한 **두 번째 보안 사고**라 회사의 내부 보안 관리 체계에 대한 의문이 제기되고 있다.

**왜 주목할만한가:** Claude Code는 Anthropic의 가장 인기 있는 제품이며, 그 경쟁력의 상당 부분이 AI 모델을 감싸는 소프트웨어 "하네스(harness)"에서 나온다. 이 하네스의 소스코드가 공개된 것이다. 개발자 입장에서는 Claude Code의 내부 동작 원리(에이전트 도구 사용, 프롬프트 구성, 보안 메커니즘 등)를 직접 들여다볼 수 있는 드문 기회이기도 하다. 동시에, AI 기업의 보안 관리 수준에 대한 업계 전반의 경각심을 높이는 사건이다.
- [Fortune](https://fortune.com/2026/03/31/anthropic-source-code-claude-code-data-leak-second-security-lapse-days-after-accidentally-revealing-mythos/) | [VentureBeat](https://venturebeat.com/technology/claude-codes-source-code-appears-to-have-leaked-heres-what-we-know)

### 미국 AI 책임법 통과 (3월)
미국에서 **AI 책임법(AI Accountability Act)**이 통과됐다. AI를 채용, 대출, 보험 등 중요 의사결정에 사용하는 기업에 대해 **정기적인 편향 감사를 실시하고 결과를 공개**할 것을 의무화한다. 올해 1월 전면 시행된 EU AI Act에 이어 미국도 본격적인 AI 규제 시대에 진입했다.

**왜 주목할만한가:** AI를 활용한 서비스를 개발하는 팀이라면, 특히 사용자 관련 의사결정에 AI를 사용하는 경우 규제 대응이 필수가 된다. EU와 미국 양쪽에서 규제가 강화되면서, AI 모델의 편향성 검증, 투명성 보고서 작성 등이 개발 프로세스에 포함되어야 할 수 있다. 글로벌 서비스를 운영하거나 계획 중인 팀은 반드시 파악해둬야 할 변화다.

- [Crescendo AI](https://www.crescendo.ai/news/latest-ai-news-and-updates)

## 2. 개발 도구 & 트렌드

### "Vibe Coding" 성숙기 진입
AI가 실시간으로 코드 생성, 제안, 리팩토링하는 흐름인 **"Vibe Coding"**이 2025년에 등장하여 2026년 본격적으로 자리잡고 있다. 핵심은 인간 대체가 아닌 **인간-AI 협업**이다. 개발자가 "분위기(vibe)"나 의도를 전달하면 AI가 코드로 변환하는 개념으로, 구체적인 코드를 한 줄씩 짜는 것이 아니라 **"무엇을 만들고 싶은지"를 설명하면 AI가 구현**하는 방식이다.

**왜 주목할만한가:** 개발 방식의 패러다임이 바뀌고 있다는 신호다. "코드를 직접 타이핑하는 개발자"에서 "AI에게 의도를 전달하고 결과를 검증하는 개발자"로 역할이 진화하고 있다. Stack Overflow 설문에서 84%가 AI 도구 사용 또는 사용 예정이라고 답한 만큼, 이 흐름에 적응하지 않으면 생산성 격차가 벌어질 수 있다. 팀 내에서 "AI와 어떻게 협업할 것인가"에 대한 가이드라인을 정할 시점이다.

> ⭐ 지금 하고 있는 CCTV 정체 판단 + ROI 자동 세팅 프로젝트가 바로 바이브코딩의 좋은 실습 사례다. "이 영상에서 차량 흐름을 분석해서 정체 구간을 판별하는 로직을 만들어줘"처럼 의도를 전달하고, AI가 OpenCV/YOLO 등을 활용한 코드를 생성하는 방식. 이 경험을 금요일 미팅에서 공유하면 팀에게도 좋은 레퍼런스가 될 수 있다.

- [CodeWeek EU](https://codeweek.eu/blog/ai-coding-tech-trends-2026/)

### Repository Intelligence — 새로운 경쟁력
단순히 코드 줄을 이해하는 것을 넘어, **코드 간의 관계와 변경 히스토리까지 파악**하는 AI가 등장하고 있다. "이 함수를 바꾸면 어디에 영향이 가는지", "이 파일은 누가, 왜 이렇게 구현했는지"를 AI가 자동으로 분석하는 방향이다. Microsoft가 2026년 주목할 7대 트렌드 중 하나로 꼽았다.

**왜 주목할만한가:** 대규모 코드베이스에서 "이 코드를 수정해도 괜찮은가?"라는 판단이 AI의 도움으로 훨씬 빠르고 정확해질 수 있다. 기존 AI 코딩 도구는 현재 파일만 보고 제안했지만, Repository Intelligence는 프로젝트 전체의 맥락과 역사를 이해하고 제안한다. 레거시 코드 유지보수나 대규모 리팩토링에서 특히 강력할 전망이다.
- [Microsoft](https://news.microsoft.com/source/features/ai/whats-next-in-ai-7-trends-to-watch-in-2026/)

### OpenRewrite, Python 지원 확대
Moderne의 자동 리팩토링 플랫폼 **OpenRewrite**가 Python 지원을 추가했다. OpenRewrite는 대규모 코드베이스에서 프레임워크 마이그레이션, 보안 패치 적용, 코드 스타일 통일 등을 **자동화**하는 도구로, 기존에는 Java/Kotlin 중심이었으나 이제 Python 프로젝트에서도 사용할 수 있게 됐다.

**왜 주목할만한가:** 수백 개 파일에 걸친 마이그레이션 작업(예: Django 버전 업그레이드, deprecated API 교체)을 수동으로 하던 것을 자동화할 수 있다. 대규모 Python 프로젝트를 관리하는 팀이라면 도입을 검토해볼 만하다.
- [DZone](https://dzone.com/articles/developer-tools-that-actually-matter-in-2026)

## 3. 커뮤니티 핫토픽

### "$500 GPU가 Claude Sonnet 코딩 벤치마크 능가" (HN)
$500짜리 GPU에서 돌린 **로컬 LLM 모델**이 Claude Sonnet 수준의 코딩 벤치마크 점수를 넘었다는 글이 Hacker News에서 화제가 됐다.

**왜 주목할만한가:** 불과 1~2년 전만 해도 로컬 모델은 클라우드 모델과 비교가 안 됐지만, 그 격차가 빠르게 좁혀지고 있다. API 비용 없이 자체 하드웨어에서 "쓸만한 수준"의 코딩 AI를 돌릴 수 있게 된다면, 데이터 보안이 중요한 환경이나 오프라인 환경에서의 활용도가 크게 높아진다. Ollama 등을 통해 직접 테스트해보면 현재 수준을 가늠할 수 있다.

- [Hacker News](https://news.ycombinator.com/front)

### LiteLLM 악성코드 공격 실시간 대응기 (HN)
LiteLLM(여러 AI 모델 API를 프록시하는 도구)에 대한 악성코드 공격을 **분 단위로 대응한 실시간 후기**가 큰 관심을 받았다. 공격 발견부터 대응, 피해 범위 파악까지의 과정을 생생하게 기록했다.

**왜 주목할만한가:** AI 도구의 공급망 공격이 점점 잦아지고 있다는 것을 보여주는 사례다. 3/31의 CanisterWorm, 이 LiteLLM 공격 등이 연달아 발생하고 있어, AI/개발 도구의 의존성 관리와 보안 점검이 일상적인 업무가 되어야 한다는 경각심을 준다.
- [Hacker News](https://news.ycombinator.com/front)

### Cocoa-Way — macOS에서 Linux 앱 네이티브 실행
macOS용 **Wayland 컴포지터**를 구현하여 Linux GUI 앱을 macOS에서 네이티브처럼 실행하는 프로젝트다.

**왜 주목할만한가:** macOS에서 Linux 전용 도구를 써야 할 때 VM이나 Docker 없이 바로 실행할 수 있다면 개발 환경 구성이 훨씬 간편해진다. 아직 초기 단계지만, 크로스 플랫폼 개발 환경의 새로운 가능성을 보여준다.

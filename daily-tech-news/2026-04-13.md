## 1. AI / LLM

### OpenAI o3-mini 기본 추론 모델로 교체 + Flex Compute 출시
OpenAI가 ChatGPT Plus 구독자의 기본 추론 모델을 **o1-mini에서 o3-mini로 교체**했다. 수학/과학 작업에서 동일 이상의 품질을 유지하면서 속도가 **3배 빨라졌다**. 동시에 **Flex Compute** 요금제도 출시 — o3를 비피크 시간대에 **30% 할인**으로 사용할 수 있는 옵션이다.

**왜 주목할만한가:** "같은 품질인데 3배 빠름"은 실무에서 체감이 큰 변화다. 추론 모델(수학적 사고가 필요한 작업)을 API로 쓰는 팀이라면 o3-mini로 전환하는 것이 합리적. Flex Compute는 비용 민감한 팀에게 좋은 선택지가 된다.
- [LLM Stats](https://llm-stats.com/ai-news)

### Arcee Trinity — 400B 오픈소스 모델, Apache 2.0
Arcee가 **400B 파라미터** 규모의 모델 Trinity를 **Apache 2.0** 라이선스로 출시했다. 스타트업이 클라우드 의존 없이 온프레미스에서 대규모 모델을 운영할 수 있도록 하겠다는 목표다.

**왜 주목할만한가:** 400B급 모델이 Apache 2.0로 나온 것은 이례적이다. 데이터 보안이 중요한 환경에서 GPT/Claude API를 대체할 수 있는 강력한 선택지. 단, 실행에 필요한 하드웨어 요구사항이 높을 것이다.
- [Crescendo AI](https://www.crescendo.ai/news/latest-ai-news-and-updates)

### OpenAI 로컬 실행용 오픈웨이트 모델 2종 출시
OpenAI가 **노트북/PC에서 실행 가능한** 소형 오픈웨이트 모델 2종을 출시했다. 개발자가 로컬에서 파인튜닝 및 추론을 자유롭게 할 수 있다.

**왜 주목할만한가:** OpenAI까지 오픈웨이트 + 로컬 실행 트렌드에 합류했다는 신호. 기존에는 API만 제공하던 OpenAI가 방향을 전환한 것. Gemma 4, Mistral, Qwen과 함께 로컬 LLM 선택지가 더 넓어졌다.

### NVIDIA GTC 2026 — 에이전틱 AI 프레임워크 중심
NVIDIA의 연례 GPU 기술 컨퍼런스(GTC 2026)가 **벤치마크 발표에서 실제 엔터프라이즈 배포**로 무게중심이 이동했다. 에이전틱 AI 프레임워크인 **NeMoCLAW와 OpenCLAW**가 역대 최다 참석 세션을 기록. 에이전트 오케스트레이션이 핵심 주제였다.

**왜 주목할만한가:** NVIDIA가 "GPU 파는 회사"에서 "AI 에이전트 인프라 회사"로 전환하고 있다는 강력한 신호. 에이전틱 AI가 더 이상 실험이 아니라 엔터프라이즈 프로덕션 수준으로 진입하고 있다.

## 2. Claude Code 업데이트

### Claude Code 주요 업데이트 — Focus View, 보안 패치, ant CLI
Claude Code가 대규모 업데이트를 받았다:

- **Focus View (Ctrl+O)** — 프롬프트, 도구 요약(edit diffstat 한 줄), 최종 응답만 보여주는 집중 모드. 복잡한 작업 중 핵심만 보고 싶을 때 유용
- **Bash 도구 권한 우회 보안 패치** — 백슬래시 이스케이프된 플래그가 읽기 전용으로 자동 허용되어 임의 코드 실행이 가능했던 취약점 수정
- **ant CLI** — Anthropic의 새 CLI 도구. Claude API를 터미널에서 직접 호출, YAML로 API 리소스 버저닝, Claude Code와 네이티브 통합
- **Web Fetch 도구 베타 출시** — Claude가 웹 페이지와 PDF의 전체 콘텐츠를 가져올 수 있는 도구
- 기타: /resume 안정성 개선, MCP 도구 개선, 에이전트/이미지/완성 워크플로우 업데이트

**왜 주목할만한가:** Focus View는 바이브코딩할 때 불필요한 출력을 줄여주는 실용적 기능. Bash 보안 패치는 즉시 업데이트해야 할 사안 — `claude update` 실행 권장.
- [Releasebot](https://releasebot.io/updates/anthropic/claude-code)

### Claude Messages API — Amazon Bedrock 리서치 프리뷰
Claude Messages API가 **Amazon Bedrock**에서 리서치 프리뷰로 사용 가능해졌다. AWS 관리 인프라에서 운영자 접근 없이 실행된다.

**왜 주목할만한가:** AWS를 사용하는 팀이라면 Anthropic API 키 없이 Bedrock을 통해 Claude를 사용할 수 있게 됐다. AWS IAM 기반 인증으로 통합이 더 쉬워진다.

## 3. 보안

### Adobe Acrobat Reader 긴급 패치 (CVE-2026-34621, CVSS 8.6)
Adobe Acrobat Reader에서 **실제 악용 중인** 치명적 취약점이 발견됐다. Prototype Pollution 기반 원격 코드 실행 가능. 즉시 업데이트 필요.
- [The Hacker News](https://thehackernews.com/)

### WordPress Smart Slider 3 Pro — 공급망 공격
업데이트 시스템이 하이재킹되어 **백도어가 포함된 v3.5.1.35가 배포**됐다. 4/7 출시 후 약 6시간 만에 탐지됐지만, 해당 시간대에 업데이트한 사이트는 원격 접근 도구가 설치됐을 가능성.

**왜 주목할만한가:** npm에 이어 WordPress 플러그인까지. 공급망 공격이 모든 패키지 생태계로 확산되고 있다. 자동 업데이트 설정을 재검토하고, 중요 업데이트는 수동 검증 후 적용하는 것이 안전하다.

### Chrome 146 — Device Bound Session Credentials (DBSC)
Google이 Chrome 146에 **DBSC 기능을 추가**했다. 하드웨어 키를 사용해 세션 쿠키 탈취를 차단한다. 인포스틸러 악성코드가 쿠키를 훔쳐도 다른 기기에서 사용 불가.

**왜 주목할만한가:** 개발자라면 Chrome 업데이트를 미루지 마세요. 특히 GitHub, 클라우드 콘솔 등의 세션 보안이 강화된다.

### Marimo Python 노트북 — 공개 9시간 만에 취약점 악용 (CVE-2026-39987)
오픈소스 Python 노트북 도구 Marimo의 인증 없는 원격 코드 실행 취약점이 공개 후 **9시간 만에 실제 공격에 사용**됐다.

**왜 주목할만한가:** 개발 도구의 취약점이 공개되면 악용까지의 시간이 갈수록 짧아지고 있다. 개발 도구도 프로덕션 서비스처럼 보안 관리가 필요한 시대.

EU AI Act이 전면 시행되면서 EU에 배포되는 모든 AI 시스템이 **투명성, 안전성, 리스크 분류** 요건을 충족해야 한다. 고위험 AI 애플리케이션은 상세 로그 유지 + 적합성 평가 통과가 필수.

**왜 주목할만한가:** AI 기반 서비스를 EU에 배포할 계획이 있다면 반드시 확인. CCTV 영상 분석 같은 것이 "고위험"으로 분류될 가능성도 있어 사전 검토 필요.

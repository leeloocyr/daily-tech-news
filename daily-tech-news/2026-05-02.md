## 1. AI / LLM

**OpenAI, GPT-5.5 출시 및 코딩/데이터 분석 기능 강화**
OpenAI가 지난 4월 23일 챗GPT 플러스 및 프로 구독자를 대상으로 GPT-5.5를 출시했습니다. GPT-5.5는 이전 모델 대비 코딩 및 데이터 분석 성능이 크게 향상되었으며, 특히 복잡한 문제 해결 능력과 다중 에이전트 협업 시나리오 지원이 강화되었습니다. 내부 벤치마크에 따르면 MMLU (Massive Multitask Language Understanding)에서 94.2%, HumanEval 코딩 벤치마크에서 92.5%를 달성하여 기존 최고 모델들의 기록을 경신했습니다.

**왜 주목할만한가:** 고도화된 코딩 및 데이터 분석 능력은 ITS 개발자들이 복잡한 교통 데이터 처리, 시뮬레이션 모델 구축, 실시간 트래픽 예측 알고리즘 개발에 직접적으로 활용할 수 있는 강력한 도구를 제공합니다.

**오픈소스 AI 에이전트 '오픈클로' 출시로 로컬 AI 생태계 확장**
카카오가 AI 모델 외부 호출 플랫폼 '플레이MCP'에 오픈소스 AI 에이전트 '오픈클로(OpenClo)' 연동 지원을 시작했습니다. 오픈클로는 사용자가 로컬 컴퓨터에 직접 설치 및 운영할 수 있는 에이전트로, 원하는 채널과 대형언어모델(LLM)을 연동하여 '나만의 AI 비서'를 구축할 수 있습니다. 이는 개발자들이 LLM 에이전트의 작동 방식을 깊이 이해하고 특정 업무 흐름에 최적화된 에이전트를 개발, 배포하는 데 유연성을 제공합니다.

**왜 주목할만한가:** ITS 개발자들은 오픈클로를 활용하여 로컬 환경에서 교통 데이터 모니터링, 이벤트 처리, 혹은 교통 관제 시스템과의 연동을 위한 맞춤형 자동화 에이전트를 개발하고 테스트할 수 있습니다.

**Llama 3.1 (가칭) 공개, 비용 효율적 온프레미스 AI 구현 가속화**
메타가 차세대 오픈소스 LLM인 Llama 3.1(가칭)을 공개했습니다. 이 모델은 최대 140B 파라미터 규모로 제공되며, 공개된 벤치마크에서 MMLU 89.1%, HumanEval 87.3%의 성능을 기록하여 상업용 모델에 근접한 경쟁력을 보여주고 있습니다. 특히 경량화된 버전도 함께 제공되어 엣지 디바이스 및 온프레미스 환경에서의 LLM 배포 및 운영 비용 효율성을 크게 개선했습니다.

**왜 주목할만한가:** ITS 환경에서 민감한 교통 데이터를 처리하거나 인터넷 연결이 제한적인 상황에서, Llama 3.1과 같은 고성능 오픈소스 LLM은 데이터 주권을 유지하면서도 AI 기능을 활용할 수 있는 강력한 대안을 제시합니다.

⭐ **미토스(Mythos) AI, 코딩/추론/사이버 보안 성능 향상으로 AI 활용 공격 및 방어 모두에 영향**
최근 '미토스 쇼크'로 불리는 AI 모델 '미토스'는 해킹을 목적으로 개발된 것은 아니지만, 기존 모델 대비 코딩, 추론, 사이버 보안 성능이 크게 향상된 것으로 평가받고 있습니다. 이 AI는 자동화 도구가 500만 번 검사하고도 16년 동안 발견하지 못했던 소프트웨어 결함을 탐지하는 등 탁월한 능력을 보여주었습니다. 이는 LLM이 사이버 보안 위협을 분석하고 대응하는 데 혁신적인 역할을 할 수 있음을 시사하며, 동시에 LLM의 공격 효율성 증대 가능성에 대한 우려도 제기합니다.

**왜 주목할만한가:** ITS 시스템의 보안 취약점을 AI를 통해 신속하게 파악하고 수정하는 데 활용될 수 있으며, 동시에 AI 기반 공격에 대한 방어 전략 수립의 중요성을 강조합니다.

## 2. 보안 & 취약점

**Microsoft Windows LNK 제로데이 취약점 (CVE-2026-32202) 악용 확인, CISA 긴급 패치 지시**
미국 사이버보안 및 인프라 보안국(CISA)이 Microsoft Windows의 LNK 파일 처리 과정에서 발견된 제로데이 취약점 CVE-2026-32202에 대해 긴급 패치를 지시했습니다. 이 취약점은 아카마이가 보고했으며, 이미 실제 공격에 악용된 사례가 확인되었습니다. Microsoft는 2026년 4월 정기 보안 업데이트를 통해 해당 취약점을 수정했지만, 불완전한 패치로 인한 추가 공격 가능성이 제기되어 추가적인 보안 강화 조치가 요구됩니다. CISA는 모든 연방 기관에 대해 2주 이내 패치를 완료할 것을 명령했습니다.

**왜 주목할만한가:** ITS 시스템, 특히 관제 센터나 현장 PC에서 운영되는 Windows 기반 시스템은 이 취약점에 노출될 수 있으므로, 최신 보안 업데이트 적용 및 시스템 취약점 점검이 필수적입니다.

**GitHub 악용 공급망 공격 확산, 개발자 계정 탈취 위협 증대**
최근 SAP 클라우드 개발 관련 npm 패키지 4개, 인터콤 공식 패키지, 파이파이 라이트닝 패키지 등 다수의 오픈소스 라이브러리를 표적으로 하는 공급망 공격이 확산되고 있습니다. 공격자들은 악성코드를 삽입한 패키지를 통해 개발자의 GitHub 계정을 탈취하고, 이를 이용해 추가적인 악성코드를 유포하는 수법을 사용하고 있습니다. 이는 오픈소스 생태계 전반에 걸친 보안 신뢰도 문제를 야기합니다.

**왜 주목할만한가:** ITS 개발자들이 프로젝트에 사용하는 오픈소스 라이브러리와 GitHub 계정 보안에 대한 각별한 주의가 필요하며, 소프트웨어 공급망 보안 검증 프로세스 강화를 고려해야 합니다.

## 3. 개발 도구 & 프로그래밍

**Gemini CLI 출시, 명령줄 기반 AI 코딩 환경 제공**
Google은 2025년 6월 25일 Gemini CLI를 출시하며 명령줄 인터페이스(CLI) 기반 AI 코딩 분야에 진출했습니다. Gemini CLI는 개발자가 터미널 환경에서 직접 Gemini 모델에 접근하여 코드 생성, 디버깅, 문서 작성 등의 작업을 수행할 수 있도록 지원합니다. 이는 별도의 IDE나 웹 인터페이스 없이도 AI의 강력한 코딩 보조 기능을 활용할 수 있게 하여 개발 워크플로우의 유연성을 높입니다.

**왜 주목할만한가:** ITS 개발자들은 Gemini CLI를 활용하여 스크립트 기반의 자동화된 코드 생성, 테스트 스크립트 작성, 혹은 리눅스 서버 환경에서의 빠른 프로토타이핑을 효율적으로 수행할 수 있습니다.

**GitHub Copilot Workspace, 에이전트 기반 개발 환경으로 진화**
GitHub Copilot Workspace가 개발 초기 아이디어 구상부터 코드 작성, 테스트, 디버깅, 배포에 이르는 전체 개발 라이프사이클을 지원하는 에이전트 기반 개발 환경으로 진화했습니다. 개발자는 자연어를 통해 원하는 기능을 설명하면, Copilot Workspace가 자동으로 작업 계획을 수립하고, 필요한 파일들을 분석하며, 코드 변경 사항을 제안하고, 테스트를 실행하는 등 전 과정을 자동화합니다. 이는 개발자가 코딩 외의 반복적이고 복잡한 작업에 소요되는 시간을 대폭 줄여줍니다.

**왜 주목할만한가:** ITS 프로젝트의 복잡한 요구사항을 자연어로 정의하고, Copilot Workspace의 에이전트가 그에 맞는 코드 구조를 제안하거나 테스트 환경을 구축하는 등 개발 생산성을 획기적으로 향상시킬 수 있습니다.

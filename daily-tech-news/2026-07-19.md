## 1. AI / LLM

*   **Moonshot AI, 오픈소스 LLM 'Kimi K3' 가중치 공개 임박, 최고급 모델 성능 추월 주장**
    중국 Moonshot AI는 7월 27일 오픈소스 LLM 'Kimi K3'의 가중치를 공개할 예정입니다. Kimi K3는 일부 작업에서 OpenAI의 GPT-5.6-Sol 및 Anthropic의 Fable 5와 같은 최상위 모델들을 능가하는 성능을 보였다고 강조되고 있습니다. 특히, 복잡한 추론 및 코드 생성 벤치마크(예: HumanEval, GSM8K)에서 높은 점수를 기록했습니다.
    **왜 주목할만한가:** 오픈소스 모델이 클로즈드 소스 최상위 모델에 근접하거나 능가하는 성능을 보이는 것은 LLM 개발 비용 절감 및 접근성 확대에 큰 영향을 미칩니다. ITS 개발자는 Kimi K3를 활용하여 온프레미스 환경 또는 특정 규제 준수가 필요한 시스템에 효율적인 AI 기능을 통합할 수 있습니다.

*   **월러스(Wallaroo), AI 에이전트용 영구 메모리 인프라 공식 출시**
    월러스가 AI 에이전트가 여러 세션에 걸쳐 컨텍스트와 학습 내용을 유지할 수 있도록 돕는 영구 메모리 인프라를 공식 출시했습니다. 이 인프라는 클로드(Claude 3.5 Sonnet), 챗GPT(GPT-4o), 제미나이(Gemini 1.5 Pro) 등 주요 생성형 AI 서비스와 통합 가능하며, 최대 1PB까지 확장 가능한 메모리 용량을 제공합니다.
    **왜 주목할만한가:** AI 에이전트의 '기억'은 복잡한 ITS 환경에서 차량 간 통신, 교통 흐름 최적화 에이전트 등의 성능을 획기적으로 향상시킬 수 있습니다. 지속적인 학습과 의사결정을 위해 영구 메모리 솔루션은 필수적인 기반 기술입니다.

*   **오라클, '퓨전 애플리케이션용 AI 에이전트 스튜디오' 고도화 발표**
    오라클이 Fusion Application용 AI 에이전트 스튜디오를 업데이트하며, 자연어 기반의 노코드/로우코드 개발 환경과 전문 개발자를 위한 프로코드 기능을 강화했습니다. 이 스튜디오는 에이전트 빌드를 위한 GitHub 저장소를 제공하며, 자사 AI 데이터 플랫폼 및 서드파티 에이전트와의 상호 호환성을 지원하여 에이전트 간의 협업 및 통합을 용이하게 합니다.
    **왜 주목할만한가:** 복잡한 ITS 시스템에 특화된 에이전트를 개발할 때, 노코드/로우코드 옵션은 빠른 프로토타이핑을 가능하게 하고, 프로코드 기능은 세밀한 제어와 커스터마이징을 제공합니다. GitHub 연동은 개발 협업과 버전 관리에 필수적입니다.

*   **AI 코딩의 새 화두 '루프 엔지니어링' 부상, 에이전트 기반 자동화 가속**
    AI 코딩 분야에서 '루프 엔지니어링(Loop Engineering)'이 새로운 화두로 떠오르고 있습니다. 이 개념은 AI가 단일 작업을 넘어 여러 단계의 피드백 루프를 통해 코드를 작성하고 검증하며, 필요시 외부 도구를 직접 다루도록 하는 것입니다. '서브에이전트' 모델을 활용하여 작성자와 검증자 역할을 분담함으로써, 코드 품질과 복잡한 문제 해결 능력을 향상시키는 데 초점을 맞춥니다.
    **왜 주목할만한가:** ITS 개발 환경에서 복잡한 센서 데이터 처리 로직, 실시간 교통 제어 알고리즘 등을 AI가 자율적으로 생성하고 최적화하는 데 루프 엔지니어링 개념이 활용될 수 있습니다. 이는 개발 생산성을 극대화하고 반복적인 코드 작업을 자동화하는 데 기여합니다.

## 2. 보안 & 취약점

*   **소닉월 SMA 제로데이 취약점 (CVE-2026-15409, CVE-2026-15410) 악용 확인**
    보안업체 래피드7은 SonicWall Secure Mobile Access (SMA) 어플라이언스의 두 가지 제로데이 취약점(CVE-2026-15409, CVE-2026-15410)이 이미 패치 공개 이전에 공격자들에 의해 활발히 악용되었음을 밝혔습니다. 이 취약점들은 기업 네트워크 침투 및 랜섬웨어 배포에 사용되었으며, 심각도(CVSSv3)는 각각 9.8 (Critical) 및 9.0 (Critical)으로 평가됩니다.
    **왜 주목할만한가:** ITS 인프라는 외부 접근이 가능한 경우가 많고, 원격 접근 솔루션의 취약점은 전체 시스템 마비를 초래할 수 있습니다. 즉각적인 패치 적용과 더불어, 제로데이 공격에 대비한 네트워크 모니터링 및 침입 탐지 시스템 강화가 필수적입니다.

*   **마이크로소프트 7월 패치 공개, 윈도우 비트락커 보안 우회 취약점 (CVE-2026-50661) 포함**
    마이크로소프트가 7월 정기 패치에서 총 570건의 취약점을 해결했으며, 이 중 윈도우 비트락커(BitLocker)의 보안 기능을 우회할 수 있는 제로데이 취약점 CVE-2026-50661이 포함되어 있습니다. 이 취약점은 공격자가 기기에 물리적으로 접근할 경우 암호화된 데이터에 접근할 수 있게 하며, 중요도(CVSSv3)는 7.8 (High)로 분류됩니다.
    **왜 주목할만한가:** ITS 환경에서 임베디드 시스템, 차량 내 컴퓨터, 엣지 디바이스 등에 비트락커와 같은 암호화 솔루션이 사용될 수 있습니다. 물리적 접근을 통한 데이터 유출 가능성은 민감한 교통 데이터 및 시스템 제어 정보의 보안에 직접적인 위협이 됩니다.

*   **개발자 GitHub 저장소 자격증명 유출 사고 빈번, 보안 강화 필요성 대두**
    최근 여러 기업에서 개발자가 GitHub와 같은 코드 저장소에 인증 토큰, API 키 등 중요한 자격증명(credential)을 함께 업로드하여 대규모 개인정보 유출 사고가 발생하고 있습니다. 이는 단순한 실수로 시작되지만, 공격자에게 시스템 접근 권한을 제공하는 심각한 보안 문제로 이어집니다.
    **왜 주목할만한가:** ITS 개발 과정에서 센서 데이터 접근 API 키, 클라우드 자원 접근 토큰 등이 GitHub에 실수로 노출될 위험이 있습니다. Git Secret, HashiCorp Vault와 같은 시크릿 관리 도구 사용 및 CI/CD 파이프라인에서 자동화된 시크릿 스캐닝 도입이 시급합니다.

## 3. 개발 도구 & 프로그래밍

*   **레저(Ledger), AI 에이전트용 지갑 보안 확장 및 개발자 도구 공개**
    하드웨어 지갑 제조업체 레저(Ledger)가 AI 에이전트의 보안을 강화하기 위한 지갑 보안 확장 기능과 개발자 도구를 공개했습니다. 이 솔루션은 '인간 승인(Human Approval)' 원칙을 유지하며, 암호화폐를 넘어 GitHub, Discord 등 범용 계정 보안까지 확장하여 AI 에이전트가 안전하게 다양한 웹 서비스를 이용할 수 있도록 지원합니다. 개발자 도구는 Ledger Live SDK v2.10.0을 통해 제공됩니다.
    **왜 주목할만한가:** ITS 시스템의 여러 구성 요소(예: 블록체인 기반 차량 ID, 스마트 컨트랙트를 통한 요금 정산, 분산원장기술 기반 데이터 공유)에 AI 에이전트가 접근할 때 강력한 보안이 필수적입니다. 레저의 솔루션은 에이전트의 자율성과 보안 사이의 균형을 제공할 수 있습니다.

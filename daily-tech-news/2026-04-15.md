### GPT-6 임박 — 코드명 "Spud", Q2 출시 예상
OpenAI의 차세대 모델 **GPT-6**(코드명 Spud)이 3월 24일에 사전 학습(pre-training)을 완료했다. Sam Altman이 "몇 주 안에 출시"라고 언급했으며, Polymarket에서는 **4월 말까지 출시 78%**, 6월 말까지 95% 확률로 예측 중. 아직 공식 출시일은 미정이지만 4-5월 중 발표가 유력하다.

**왜 주목할만한가:** GPT-5.4가 나온 지 한 달 반 만에 GPT-6이 임박했다는 건 모델 발전 속도가 더 빨라지고 있다는 의미. Claude Mythos, Gemini 3.1 Pro와의 경쟁이 한층 치열해질 전망. GPT-6이 나오면 모델 선택 전략을 다시 재검토해야 할 수 있다.

## 2. Claude Code — Routines 기능 출시 (리서치 프리뷰)

### Claude Code Routines — 반복 자동화의 새로운 형태
Anthropic이 **Claude Code Routines**를 리서치 프리뷰로 출시했다. **프롬프트 + 레포 + 커넥터를 한 번 설정하면, 스케줄/API/이벤트에 의해 자동 실행**되는 기능이다.

**기존 기능과의 차이:**
| 기능 | 실행 환경 | 오프라인 | 주기적 | 레포 접근 |
|---|---|---|---|---|
| `/loop` | 내 터미널 | ❌ | ✅ | ✅ (로컬) |
| `/schedule` | 클라우드 | ✅ | ✅ | GitHub만 |
| **Routines** | **클라우드** | **✅** | **✅** | **✅ (레포+커넥터)** |

**핵심 차이:** Routines는 `/schedule`과 `/loop`의 장점을 합친 것. 클라우드에서 돌아서 노트북 꺼도 OK, 레포와 커넥터에 접근 가능, 스케줄/API/이벤트 트리거 모두 지원.

**플랜별 일일 실행 한도:**
| 플랜 | 일일 한도 |
|---|---|
| Pro | 5회 |
| Max | 15회 |
| Team | 20회 |
| Enterprise | 25회 |

**사용 사례:**
- 매일 밤 버그 자동 분류
- 팀 체크리스트 기반 자동 코드 리뷰
- 언어 간 변경사항 자동 포팅
- 배포 후 에러 자동 체크

**왜 주목할만한가:** 지난번에 "스케줄은 로컬 파일 접근 안 되고, /loop은 터미널 닫으면 죽는다"고 했었는데, Routines는 **클라우드 실행 + 레포/커넥터 접근**으로 두 가지 한계를 모두 해결했다. GitHub 레포를 연결하면 코드 변경도 가능하니, 이전에 우리가 포기했던 "매일 아침 자동 뉴스 수집" 같은 것도 Routines로 구현 가능할 수 있다.

- [Claude Blog](https://claude.com/blog/introducing-routines-in-claude-code) | [9to5Mac](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/) | [The Register](https://www.theregister.com/2026/04/14/claude_code_routines/)

## 4. 개발 트렌드

### DORA 2025 보고서 — "AI는 자동 성과 향상이 아님"
Google의 DORA(DevOps Research and Assessment) 보고서에 따르면, **AI가 소프트웨어 배포 성과를 자동으로 향상시키지는 않는다**. AI는 기존 엔지니어링 역량의 **배율기(multiplier)** 역할을 할 뿐, 기반이 탄탄하지 않으면 효과가 없다.

**왜 주목할만한가:** "AI 도입하면 자동으로 좋아진다"는 환상에 대한 데이터 기반 반론. AI 도구를 도입하기 전에 팀의 기본 엔지니어링 문화(CI/CD, 테스트, 코드 리뷰)를 먼저 갖추는 것이 중요하다는 메시지.

### Meta — AI로 대규모 데이터 파이프라인의 "암묵 지식" 매핑
Meta가 AI 에이전트를 사용해 대규모 데이터 파이프라인에서 **문서화되지 않은 암묵적 지식(tribal knowledge)**을 자동 매핑하는 사례를 공개. AI가 코드만 보고 컨텍스트 없이 작업하면 "컴파일은 되지만 미묘하게 잘못된" 코드를 생성하는 문제를 해결하기 위한 시도.

**왜 주목할만한가:** 우리가 CLAUDE.md를 만드는 것도 같은 맥락이다. AI에게 도메인 지식(ITS 용어, 코딩 컨벤션, 시스템 특성)을 명시적으로 알려주지 않으면 "컴파일은 되지만 틀린" 결과가 나온다.
- [Meta Engineering](https://engineering.fb.com/2026/04/06/developer-tools/how-meta-used-ai-to-map-tribal-knowledge-in-large-scale-data-pipelines/)

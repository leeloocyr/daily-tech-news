## 1. AI / LLM

### Google TurboQuant — LLM 메모리 효율 혁신 (ICLR 2026)
Google 연구팀이 ICLR 2026에서 **TurboQuant** 알고리즘을 발표했다. LLM의 KV 캐시(대화 맥락을 저장하는 메모리)가 차지하는 메모리를 크게 줄이는 기술로, PolarQuant 벡터 회전 + Johnson-Lindenstrauss 압축을 결합한 2단계 방식이다. 이를 통해 100만 토큰급 긴 컨텍스트 모델이 훨씬 적은 메모리로 동작할 수 있게 된다.

**왜 주목할만한가:** 1M 토큰 컨텍스트가 보편화되면서 메모리 비용이 핵심 병목이 됐는데, TurboQuant은 이 문제를 직접 해결한다. 같은 하드웨어에서 더 긴 컨텍스트를 처리할 수 있게 되므로, 엣지 디바이스에서 AI를 돌리는 것도 더 현실적이 된다.

> ⭐ CCTV 영상 분석처럼 연속적인 프레임을 처리할 때, 이전 프레임들의 맥락을 유지하면서 분석하는 것이 중요한데, KV 캐시 메모리가 줄어들면 엣지 디바이스에서 더 긴 시간 동안의 교통 패턴을 추적할 수 있게 된다.

- [LLM Stats](https://llm-stats.com/ai-news)

### Apple CarPlay + ChatGPT 음성 통합
OpenAI가 ChatGPT를 **Apple CarPlay에 통합**했다. iOS 26.4 이상에서 CarPlay 화면에서 ChatGPT와 음성 대화가 가능하다. 차량/앱 제어는 불가하고 순수 음성 대화만 지원되며, wake word 없이 앱을 직접 열어야 한다. 모든 ChatGPT 플랜에서 글로벌로 사용 가능.

**왜 주목할만한가:** AI가 차량 대시보드에 들어온 것이 중요한 신호다. 아직은 단순 대화만 가능하지만, 향후 Siri AI 버전과 결합되면 네비게이션, 차량 제어까지 AI가 담당하게 될 것이다. AI 음성 인터페이스가 모바일을 넘어 자동차로 확장되는 흐름.
- [MacRumors](https://www.macrumors.com/2026/03/31/openai-chatgpt-carplay/) | [Engadget](https://www.engadget.com/ai/openai-brings-chatgpts-voice-mode-to-carplay-191422297.html)

### Anthropic IPO 계획 — 10월 목표, $400-500B 밸류에이션
Anthropic이 **2026년 10월 IPO**를 목표로 하고 있으며, $400-500B(약 560-700조 원) 밸류에이션에 $60B 이상 자금 조달을 계획하고 있다는 보도가 나왔다.

## 2. Anthropic — Claude 서비스 장애 (4/6)

### Claude 전면 장애 — 약 2시간
어제(4/6) 오전 10:30(ET)부터 약 2시간 동안 **Claude.ai 전면 장애**가 발생했다. 로그인, Voice Mode, 채팅, Claude Code 등 모든 서비스가 영향을 받았으며, Downdetector 기준 **약 5,000건**의 신고가 접수됐다. 12:44(ET)에 복구 완료.

**왜 주목할만한가:** 지난주 소스코드 유출, 서드파티 과금 변경, 정부 블랙리스트에 이어 서비스 장애까지 — Anthropic에게 힘든 한 주였다. Claude에 의존하는 워크플로우가 있다면, 장애 시 대체 수단(GPT, Gemini 등)을 미리 준비해두는 것이 현실적이다.

## 3. 개발 도구 & 트렌드

### GitHub Copilot SDK — Public Preview 전환 (4/2)
GitHub Copilot SDK가 Technical Preview에서 **Public Preview**로 전환됐다. Copilot의 에이전트 런타임을 **자신의 앱에 직접 내장**할 수 있는 SDK로, Node.js/TypeScript, Python, Go, .NET, Java를 지원한다.

주요 기능: 도구 호출, 스트리밍, 파일 조작, 멀티턴 세션이 기본 제공되며, 도메인별 커스텀 도구와 에이전트를 정의할 수 있다. 시스템 프롬프트도 replace/append/prepend 방식으로 커스터마이즈 가능.

**왜 주목할만한가:** 이전까지 AI 에이전트를 앱에 넣으려면 LangChain/LangGraph 같은 별도 프레임워크가 필요했는데, 이제 GitHub의 프로덕션 검증된 에이전트 런타임을 SDK로 바로 가져다 쓸 수 있다. OpenAI Responses API, Claude Agent SDK와 함께 "에이전트 SDK 3파전"이 형성되고 있다. 자체 서비스에 AI 에이전트를 넣고 싶은 팀이라면 비교 검토해볼 시점.
- [GitHub Blog](https://github.blog/news-insights/company-news/build-an-agent-into-any-app-with-the-github-copilot-sdk/) | [GitHub Changelog](https://github.blog/changelog/2026-04-02-copilot-sdk-in-public-preview/)

### Claude Code 업데이트 — /powerup, 성능 개선
Claude Code에 **인터랙티브 `/powerup` 레슨**이 추가됐다. Claude Code 사용법을 대화형으로 배울 수 있는 기능이다. 이 외에도 resume 기능 강화, 전반적인 성능 개선, 다이얼로그/훅/편집/스크롤/PowerShell 관련 다양한 버그 수정이 포함됐다.

**왜 주목할만한가:** `/powerup`은 Claude Code를 처음 쓰거나 팀에 도입할 때 유용한 학습 도구다. 바이브코딩을 연습하는 과정에서 활용해볼 만하다.
- [Releasebot](https://releasebot.io/updates/anthropic/claude-code)

# Edge TTS vs Web Speech API — 한국어 음성 품질 비교

**테스트 일자:** 2026-04-08
**테스트 도구:** Microsoft Edge TTS, 브라우저 Web Speech API
**프로젝트:** Daily Tech News Reader

---

## 배경
매일 정리하는 기술 뉴스를 음성으로 읽어주는 웹 앱을 만들면서, 처음에는 브라우저 내장 Web Speech API를 사용했다. 그런데 결과물이 어색한 기계음에 가까워서, 더 자연스러운 AI TTS로 교체했다.

## 비교 대상

| 항목 | Web Speech API | Edge TTS (HyunsuMultilingualNeural) | Edge TTS (InJoonNeural) |
|---|---|---|---|
| 종류 | 브라우저 내장 | Microsoft Edge AI TTS | Microsoft Edge AI TTS |
| 언어 | 다국어 | 다국어 (한국어 포함) | 한국어 전용 |
| 비용 | 무료 | 무료 (API 키 불필요) | 무료 (API 키 불필요) |
| 설치 | 불필요 | `pip install edge-tts` | `pip install edge-tts` |

## 테스트 방법
같은 뉴스 마크다운(2026-04-01.md, 약 3분 분량)을 세 가지 음성으로 읽혀서 비교.

## 결과

### 1. Web Speech API
- **음질:** ★★☆☆☆ — 명백한 기계음. 억양이 단조롭고 부자연스러움
- **속도:** 즉시 재생 (서버 처리 불필요)
- **장점:** 설치/설정 없이 바로 사용 가능, 완전 클라이언트 사이드
- **단점:** "AI 음성"이라기엔 너무 어색함. 뉴스 앵커 느낌 전혀 없음

### 2. Edge TTS — HyunsuMultilingualNeural
- **음질:** ★★★★☆ — 사람 목소리에 가까움
- **속도:** 첫 재생 시 5-10초 대기 (서버에서 mp3 생성)
- **문제점:** **숫자를 스페인어로 읽는 현상** 발생
  - "1.", "2." 같은 항목 번호를 다른 언어로 인식
  - 다국어 모델이라 한국어 컨텍스트에서도 가끔 다른 언어로 발음
- **결론:** 품질은 좋지만 한국어 뉴스에는 부적합

### 3. Edge TTS — InJoonNeural ⭐
- **음질:** ★★★★☆ — HyunsuMultilingual과 비슷한 수준
- **속도:** 첫 재생 5-10초, 캐시 후 즉시
- **장점:**
  - 한국어 전용 모델이라 **숫자도 정확하게 한국어로 읽음**
  - 자연스러운 억양, 뉴스 앵커 같은 톤
  - 무료, API 키 불필요
- **단점:**
  - 서버에서 mp3 생성 필요 (첫 재생 지연)
  - 인터넷 연결 필수 (Microsoft 서버 호출)

## 최종 선택
**Edge TTS InJoonNeural** 채택. 이유:
1. 한국어 품질이 압도적으로 자연스러움
2. 숫자 발음 문제 없음
3. 무료 + API 키 불필요로 진입장벽 낮음
4. 캐싱 적용 시 성능 문제 없음

## 미팅 공유 포인트
- 다국어 AI 모델이 항상 좋은 건 아니다 — **언어 전용 모델이 더 정확할 수 있다**
- 무료 AI TTS가 API 키 없이도 충분히 사용 가능한 수준이 되었다
- TTS는 여전히 캐싱 전략이 중요하다 (3분 분량 mp3 생성에 5-10초)
